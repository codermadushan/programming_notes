---------- First go program ----------------------------------------------------

package main

import "fmt"

func main() {
	fmt.Print("Madushan")
}


* Go allows only ONE main() function per package, and by default, all .go files
  in a folder belong to the same package (unless explicitly specified otherwise).


* In Go, if you're writing an executable program, it must:

    Use "package main"

    Have a main() function as the entry point

    If you try to run a file without "package main" or without a "main()"
    function, you’ll get an error.


* "package <name>"  (name is not main) is a library-style packages — used for
  reusable code. They don’t need a main() function, and they’re not meant to be
  executed directly with "go run".


* In this case <import "fmt"> means, we import "Formatted Input & Output package" from Standart Library of Go.

---------- Variables -----------------------------------------------------------

* Define a variable:

    var <variable_name> <variable_data_type> = <value>


    If you provide the value to the variable at the time of declaration, then
    no need to specify the datatype.

        var name = "Madushan"

        Inside a function we can further simplify this as follows, But it is
        valid only inside functions (will be formatted automatically by go extension)

            var name = "Madushan" ----> name := "Madushan"


    If you don't provide the value at the time of declaration, then the data type
    is required.

        var name string


When we create a variable without providing a value at the time of declaration,
default values of each datatype will be assigned to the variable.

  * int: 0

  * float64: 0

  * string: "" (Empty string)

  * bool: false

  * Function: var value func(string) int : nill

  * Pointer : var value *int             : nill

  * Map     : var value map[string]int   : nill

  * Slice   : var value []int            : nill

  * Struct  : var value Person           : { 0 }
    The value { 0 } is just a representation. In this case default values are applied
    to the properties of the "Person" struct.


* We can assign values to multiple variables in one line as follows:

    const name, age = "Madushan", 10


* Scope:
    A variable can be accessed only inside the block which that variable is declared.

    global variables can be accessed from anywhere within the same package.
    But not necessarily across packages unless you make them exported.

        PascalCase = Exported

        camelCase = private (within the package)


* Naming conventions
  1. PascalCase: structs, interfaces, enums

  2. snake_case: file names

  3. camelCase : variables

  4. UPPERCASE : constants

  5. lowercase : package names


* Constants:
      To prevent re-assign values to variables, use "const" instead of "var". const
      variables must be initialized at the time of declaration.

          const name string (incorrect)

          const name = "Madushan" (correct)
          const Pi = 3.14159 (correct)

          const name string = "Madushan" (correct, but no need to specify the type of the variable)

          * Define multiple consts at once (Const Block)
            const (
              const1 = 1
              const2 = 2
            )


      Always use "const" to define variables, use "var" only if you cannot use const




* variable1 = variable2

      If the variable2 is a:

          int, float, bool, string
          array
          struct

          then, new copy will be created and will be assigned to the variable1.


      Otherwise variable1 is a new reference to the variable2

---------- Loops ---------------------------------------------------------------

---------- For loop ----------

* Iterate integers from 1 to 100:

    for i := 1; i <= 100; i++ {
      fmt.Println(i)
    }

    Shortcut: "for"




* If you want to iterate integers from "0" to "n-1":

    for i := range n {
      fmt.Println(i)
    }

    Shortcut: "forr" (for range)




* Iterate over collection:

    arr := [5]int{10, 20, 30, 40, 50}


    // Access both indexes and values
    for index, value := range arr {
      fmt.Println(index, value)
    }


    // Access only indexes
    for index := range arr {
      fmt.Println(index)
    }


    // Access only values
    for _, value := range arr {
      fmt.Println(value)
    }


    // Access nothing
    for range arr {
      //
    }




* We can use "break" and "continue" statements like in other languages to control
  the flow of the loop.




---------- while loop ----------

* There is no "while" loop in go language. We can achieve it using "for" loop.

    i := 1
    for i <= 5 {
      fmt.Println(i)
      i++
    }


* To create an infinite loop:

    for {
      // body
    }


* We can use "break" and "continue" keywords to control the flow of the loop.

---------- switch ----------

* No need to use "break" at the end of each condition body. But if you want to
  without break behavior we can use "fallthrough" keyword. It will fallthrough
  the control to the next case also. but not all cases.


Ex:

    func findStr(str string) {
      switch str {
      case "hello":
        fmt.Println("str is hello")

      case "world":
        fmt.Println("str is world")

      default:
        fmt.Println("Neither hello or world")
      }
    }


* To check the type of a value, use "<value>.(type)" as the expression.

Ex:

    func findValueType(value any) {
      switch value.(type) {
      case int:
        fmt.Println("value is a int")

      case string:
        fmt.Println("value is a string")

      default:
        fmt.Println("value is neither int nor string")
      }
    }

---------- Arrays --------------------------------------------------------------

* The size of an array must be a fixed number.


* All values must be in same data type.


* We cannot use "const" to define arrays. We have to use "var".


* Declaration:

    With size:

        var <array_name> [<length_of_array>]<data_type>

        Ex: var numbers [5]int


    With values:

        <array_name> := [<length>]<data_type>{value1, value2, ...}

        Ex: numbers := [3]int{1, 2, 3}




* The data type of the array is:

      [<size>]<type_of_a_value>




* Change the value at specific index:

    <array_name>[<index>] = <new_value>

    Ex: numbers[0] = 10




* Check the length of an array:

    len(array)




* Comparing arrays:

    To compare arrays, the types of both arrays must be same.

    numbers1 := [3]int{1, 2, 3}
    numbers2 := [3]int{1, 2, 3}

    fmt.Println(numbers1 == numbers2) // true


    Note: <, <=, >, >= are not defined for arrays.




* 2D arrays:

      numbers := [3][3]int{
        {1, 2, 3},
        {4, 5, 6},
        {7, 8, 9},
      }

---------- Slices --------------------------------------------------------------

* A slice is a pointer to an underlying array. The length of a slice is not
  a fixed value like an array.


* But internally slices use arrays. To get the length of current underline array
  of a slice:

      cap(<slice>)




---------- Creation ----------

1. Without values:

  Option 1:
      var slice []int

  Option 2:
      slice = make([]int, <initialLength>)


2. With values:

    numbers := []int{20, 40, 60, 80, 100}


3. From an array:

    numbersArray := [5]int{20, 40, 60, 80, 100}

    slice1 := numbersArray[n:m]  (from index n to m-1)
    slice2 := numbersArray[:m]  (from index 0 to m-1)
    slice3 := numbersArray[n:] (from index n to last)


4. From a slice:

    numbersSlice := []int{20, 40, 60, 80, 100}

    slice1 := numbersSlice[n:m]  (from index n to m-1)
    slice2 := numbersSlice[:m]  (from index 0 to m-1)
    slice3 := numbersSlice[n:] (from index n to last)




---------- Add a new value ----------

    numbersSlice = append(numbersSlice, 10, 20)




---------- Make a copy of an existing slice ----------

* We cannot create a copy of an exising slice by assigning the existing slice to
  a new variable using = operation. In this case new variable becomes a new
  reference to existing slice.


Step 1:
    Create a new slice with the same length and type of existing slice.

    sliceCopy = make([]int, len(slice))


Step 2:
    Use copy function to copy values to new slice.

    copy(sliceCopy, slice)




---------- Access values ----------

* We can access values of a slice using indexes.

    slice[<index>]




---------- Iterating over a slice ----------

    slice := []int{20, 40, 60, 80}

    for index, value := range slice {
      fmt.Println(index, value)
    }




---------- Functions of slices ----------

* To compare the equality of 2 slices (compare length & values). Cannot compare
  2 slices in different data types.

      slices.Equal(slice1, slice2)

---------- Maps ----------------------------------------------------------------

* Map provides an efficient way to store and retrieve key value pairs.

* Each key must be unique.

* They are unordered collections. There is no guaranteed order when iterating
  over them.




---------- Creation ----------

* Without values:

      mapVariable2 := make(map[string]int)


* With values:

    mapVariable := map[string]int{
      "key1": 89,
      "key2": 26,
    }




---------- Add a new pair to existing map ----------

* In this case if the key already exists, then the existing value will be
  replaced with new value.

      mapVariable["key2"] = 3




---------- Access values ----------

* By using keys we can access values. If the key doesn't exist, then default
  value of values data type will be returned.

    Option 1:

        value := mapVariable[<key>]


    Option 2:

        value, isKeyExist := mapVariable[<key>]




---------- Delete key value pairs ----------

* If the key doesn't exist, nothing will be happened.


* Delete one pair:

      delete(<map>, <key>)


* Delete all pairs:

      clear(<map>)




---------- Map functions ----------

* maps.Equal(mapVariable1, mapVariable2):

      To check both maps contain same key value pairs or not.




---------- Iterate over a map ----------

* Order is not preserved.

      mapVariable := map[string]int{
        "key1": 1,
        "key2": 2,
        "key3": 3,
        "key4": 4,
        "key5": 5,
      }

      for key, value := range mapVariable {
        fmt.Println(key, value)
      }




---------- Get number of key value pairs in a map ----------

* len(mapVariable)

---------- Functions -----------------------------------------------------------

func <name>(<parameters>) <returnType> {
  // body

  return <value>
}


* When you pass an argument to a function new copy will be created for,

      ints, floats, bools, strings, arrays and structs.

  For other data types, parameter will becomes a reference to the original value.


* Without a return value:

      func testFunction() {
        fmt.Println("TestFunction")
      }


* All parameters in same data type:

      func add(a, b int) int {
        return a + b
      }


* Parameters in different data types:

      func getAge(name string, age int) string {
        return fmt.Sprintf("%s is %d years old", name, age)
      }


* Parameters in mixed types:

      func testFunction(a, b int, c string) {
        fmt.Println(a, b, c)
      }




---------- Anonymous functions ----------

* If we want to create a function inside another function, it must be an
  anonymous function.


* Execute immediately:

      func() {
        fmt.Println("Madushan")
      }()


* Assign to a variable:

      operation := func() {
        fmt.Println("Madushan")
      }




---------- Function as an argument ----------

func applyOperation(a, b int, operation func(int, int) int) int {
	return operation(a, b)
}

func main() {
	add := func(a, b int) int {
		return a + b
	}

	result := applyOperation(10, 20, add)

	fmt.Println(result) // 30
}




---------- Return a function from a function ----------

func createMultiplier(factor int) func(int) int {
	return func(x int) int {
		return x * factor
	}
}

func main() {
	multiplier := createMultiplier(10)

	result := multiplier(4)

	fmt.Println(result)
}




---------- Multiple return values ----------

* We must access all return values at the time of calling. Otherwise you will
  get an error.

      func getUserInfo() (string, int) {
        return "Madushan", 26
      }

      func main() {
        name, age := getUserInfo() // Must access both name and age

        fmt.Println(name, age)
      }


* Simple error handling:

      func compare(a, b int) (string, error) {
        if a < b {
          return "b is greater than a", nil
        } else if a > b {
          return "a is greater than b", nil
        }
        return "", errors.New("both values are equal")
      }

      func main() {
        result, error := compare(10, 10)

        if error != nil {
          fmt.Println("Error:", error)
        } else {
          fmt.Println(result)
        }
      }


* When we return multiple values from a function, we can variable names of
  return types. In this case specified variables are created with their zeroth
  values and we are only require to specify "return" keyword.


Ex:

    func getAddMul(a, b int) (add int, mul int) {
      add = a + b
      mul = a * b
      return
    }




---------- Variadic functions ----------

* Functions that accept variable number of arguments.


* Variable number of parameter is a "slice" and it must be the last parameter.


Ex:

    func calculateTotal(numbers ...int) int {
      total := 0

      for _, value := range numbers {
        total += value
      }

      return total
    }

    func main() {
      numbers := []int{10, 20, 30, 40, 50}

      result := calculateTotal(numbers...)  // Like spread operator

      fmt.Println(result)
    }




---------- defer keyword ----------

* defer <expression>


* "defer" expressions are executed at the very end of the function in LIFO order.


Ex:

    func main() {
      fmt.Println("Start")

      defer fmt.Println("One")
      defer fmt.Println("Two")

      fmt.Println("Middle")

      defer fmt.Println("Three")
      defer fmt.Println("Four")

      fmt.Println("End")
    }


Output:

    Start
    Middle
    End
    Four
    Three
    Two
    One




* Difer statements evaluate immediately when they are encountered. But executed
  at the very end of the function in LIFO order.


Ex:

    func main() {
      i := 10

      fmt.Println("Start")

      defer fmt.Println("value of 'i' in defer statement:", i)

      i++

      fmt.Println("Real value of 'i':", i)

      fmt.Println("End")
    }


Output:

    Start
    Real value of 'i': 11
    End
    value of 'i' in defer statement: 10




* Practical use cases of defer:

    Resource cleanup
    Unlocking mutexes
    Logging and Tracing


* Best practices:

    Keep deferred actions short
    Understand evaluation timing
    Avoid complex control flow




---------- panic keyword ----------

* panic(<anyValue>)


* Main goroutine panic stops the full program.


* Other goroutine panic stops only that goroutine, program continues.


Ex:

    func process(input int) {
      if input < 0 {
        panic("input must be a non-negative number")
      }

      fmt.Println("processing input:", input)
    }




---------- defer with panic ----------

* All the difer expressions that are evaluated before panic statement will be
  executed before panic in LIFO order.


Ex:

    func process(input int) {
      defer fmt.Println("difer statement 1")

      defer fmt.Println("difer statement 2")

      if input < 0 {
        fmt.Println("before panic")

        panic("input must be a non-negative number")
      }

      fmt.Println("processing input:", input)

      defer fmt.Println("difer statement 3")
    }

    func main() {
      process(-2)

      fmt.Println("End")
    }


Output:

    before panic
    difer statement 2
    difer statement 1
    panic: input must be a non-negative number




---------- recover ----------

* We use recover() function to recover from a panic.


* If there is not panic in the function execution, recover() returns nil,
  otherwise it returns the panic.


Ex:

    func process() {
      defer func() {
        if r := recover(); r != nil {
          fmt.Println("recovered:", r)
        }
      }()

      fmt.Println("Start of process")

      panic("Something went wrong!")

      fmt.Println("End of process")
    }

    func main() {
      fmt.Println("Start of program")

      process()

      fmt.Println("End of program")
    }


Output:

    Start of program
    Start of process
    recovered: Something went wrong!
    End of program


* Practical use cases:

    Graceful recovery
    Cleanup
    Logging & Reporting


* Best practices:

    Use with defer
    Avoid silent recovery
    Avoid overuse




---------- exit ----------

---------- init function ----------

* A special function that can be declared in any package. It is used to perform
  initialization tasks for the package before it is used.


* It doesn't have any parameter or return value.


* It will be executed before the main function.


* It will be executed only once even if we import that package multiple times.


* We can define multiple init functions in single package. They will be executed
  in their declaration order.


Ex:

    func init() {
      fmt.Println("Initialization 1")
    }

    func init() {
      fmt.Println("Initialization 2")
    }

    func main() {
    }


Output:

    Initialization 1
    Initialization 2


* Practical use cases:

    Setup tasks
    Configuration
    Registering components
    Database initialization




---------- (Custom function types) ----------

* If the custom function type method has a value receiver:

      Ex:

          type FuncType func(i, j int)

          func (ft FuncType) call(i, j int) {
            ft(i, j)
          }

          func main() {
            f := func(i, j int) {
              fmt.Println(i + j)
            }

            FuncType(f).call(10, 20)
          }


* If the custom function type method has a pointer receiver:

      Ex:

          type FuncType func(i, j int)

          func (ft *FuncType) call(i, j int) {
            (*ft)(i, j)
          }

          func main() {
            f := func(i, j int) {
              fmt.Println(i + j)
            }

            ft := FuncType(f)
            (&ft).call(10, 20)
          }

---------- Closures ------------------------------------------------------------

Ex:

    func adder() func() int {
      i := 0

      fmt.Println("Value of i:", i)

      return func() int {
        i++

        fmt.Println("Added 1 to i")

        return i
      }
    }

    func main() {
      adder1 := adder()

      fmt.Println(adder1())
      fmt.Println(adder1())

      fmt.Println("-----------------------------------")

      adder2 := adder()

      fmt.Println(adder2())
      fmt.Println(adder2())
    }


Output:

    Value of i: 0
    Added 1 to i
    1
    Added 1 to i
    2
    -----------------------------------
    Value of i: 0
    Added 1 to i
    1
    Added 1 to i
    2


* If we call "adder1" multiple times, "i" will be incremented one by one and the
  the state of "i" will be persisted in the RAM.


* "adder2" maintains a seperate state which is independent from "adder1".

        return 1

---------- Pointers ------------------------------------------------------------

* A pointer is a variable that stores the memory address of another variable.


* Type of the variable must be matched with the type of pointer.


* Go doesn't support pointer arithmetics.


* Creation:

      Option 1: Create pointer and then assign a value

          var ptr *int
          value := 10
          ptr = &value  // Referencing


      Option 2: Create pointer and assign a value together

          value := 10
          ptr := &value  // Referencing


* Access the value from a pointer variable (Dereferencing):

      *<pointerVariable>


* Update the value of a pointer variable using a function:

      func modifyInt(i *int) {
        *i++
      }

      func main() {
        value := 10

        modifyInt(&value)

        fmt.Println(value)  // 11
      }




---------- (new function) ----------

* Allocate zeroth value of the specified data type in memory and return its
  memory address (pointer).


Ex:

    intPointer := new(int)
    fmt.Println(*intPointer) // 0

---------- Strings -------------------------------------------------------------

* Strings in go are immutable.


* To consider everything as a plain string use ``. Multilines are allowed.

      `sample string`


* To support escape characters use "". Multilines are not allowed.

      "sample string"


* In go, we can consider a string as an array of runes.


* Get the length of a string:

      len(<string>)


* We can access characters (runes) in a string using indexes:

      name := "Madushan"
      fmt.Printf("%c", name[0]) // M


* We can perform string concatenation using + operator.

      str1 := "abc"
      str2 := "def"

      str := str1 + str2

      fmt.Println(str) // abcdef


* String comparison is performed character by character from start to end.

    Ex 1:

      str1 := "abc"
      str2 := "ab"
      fmt.Println(str1 > str2) // true


    Ex 2:
      str1 := "abc"
      str2 := "abd"
      fmt.Println(str1 > str2) // false


* Iterating over a string:

      for _, char := range name {
        fmt.Printf("%c", char)
      }

---------- Runes ---------------------------------------------------------------

* Runes are int32 integers represent characters.


* We use single quotes to define runes.

      char := 'a' or (var char rune = 'a')

* Convert a rune to a string:

      char := 'a'
      str := string(char)

---------- Formatting verbs ----------------------------------------------------

%%  - To print %

%T  - Type of the value

%v  - Default value




---------- Integers ----------

* Base 2:

      %b


* Base 10:

      %d

      %<number>d
          If the number of characters in the value less than specified number,
          then add spaces to left side to set the length of the value to the
          specified number. (%4d)

          If the specified number is negative, then add the spaces from right
          side. (%-4d)

          If the specified number is specified with 0, then add zeros to left
          side. (%04d)

          If the number of characters in the value greater than or equal to
          specified number, then print the normal value.



* Base 16:

      %x - Value in lowercase
      %X - Value in uppercase




---------- Floats ----------

* Decimal point:

      %f

      %.<number>f - Default width, Precision <number>

      %<number1>.<number2>f - Width <number1>, Precision <number2>


* Scientific notation with "e" as exponent:

      %e




---------- Booleans ----------

* %t




---------- Strings ----------

* Value as a plain string:

      %s

      %<number>s - Length is <number>. Right justified.

      %<-number>s - Length is <number>. Left justified.

---------- fmt package ---------------------------------------------------------

---------- Printing functions ----------

      To print one or more values.
      New line will be added at the end.
      An space will be added between each value.


* fmt.Print:

      To print one or more values.
      New line won't be added at the end.


* fmt.printf:

      To print with formatting verbs.




---------- Formatting functions ----------

* fmt.Sprint / fmt.Sprintln / fmt.Sprintf:

      Same as printing functions. But nothing will be printed. Formatted string
      will be returned.




---------- Scanning functions ----------

* To take inputs from the user.


* fmt.Scan:

      var name string
      var age int

      fmt.Print("Enter your name and age:")
      fmt.Scan(&name, &age)

      fmt.Printf("Name: %s, Age: %d", name, age)


      We can enter inputs by seperating them using spaces or new lines.


* fmt.Scanln:

      Simillar to fmt.Scan. But inputs can be seperated only using spaces.


* fmt.Scanf:

      To get inputs in specific format

          var name string
          var age int

          fmt.Print("Enter your name and age:")
          fmt.Scanf("%s %d", &name, &age)

          fmt.Printf("Name: %s, Age: %d", name, age)




---------- Error formatting functions ----------

* fmt.Errorf:

      To format error messages.

      Ex:
          func checkAge(age int) error {
            if age < 18 {
              return fmt.Errorf("age %d is too young to drive", age)
            }
            return nil
          }

---------- Structs -------------------------------------------------------------

* Structs in go are composite data types that allow you to group together
  defferent type of variables under single name.


* They are simillar to classes in object oriented languages. But they do not
  support inheritance.


* Creation:

      type <name> struct {
        <property1> <dataType>
        <property2> <dataType>
        ...
      }

      Ex:
          type Person struct {
            name string
            age  int
          }


* Initialization:

      person := Person {
        name: "Madushan",
        age:  26,
      }

      If we omit the initialization of any field, the default value of that data
      type will be assigned to that field.


* Access fields:

      p := Person {
        name: "Madushan",
        age:  26,
      }

      fmt.Printf("Name: %s, Age: %d", p.name, p.age)


* Update fields:

      p := Person {
        name: "Madushan",
        age:  26,
      }

      p.age = 30


* Anonymous structs:

      user := struct {
        userName string
        email    string
      } {
        userName: "codermadushan",
        email:    "codermadushan@gmail.com",
      }




---------- Struct methods ----------

* We can define methods for structs in go. But we cannot define them inside the
  struct.


* We can create structs inside functions. But if you want to create methods for
  a struct, that struct must be defined in package level. Because we cannot
  create struct methods inside any function.


Ex: (With value receiver)

    type Person struct {
      firstName string
      lastName  string
      age       int
    }

    func (p Person) fullName() string {
      return p.firstName + " " + p.lastName
    }

    func main() {
      p := Person{
        firstName: "Madushan",
        lastName:  "Wijewardhana",
        age:       26,
      }
      fmt.Println(p.fullName())
    }


Ex: (With pointer receiver)

    type Person struct {
      name string
      age  int
    }

    func (p *Person) incrementAge() {
      p.age++ // same as (*p).age++
    }

    func main() {
      p := Person{
        name: "Madushan",
        age:  26,
      }

      p.incrementAge()

      fmt.Println(p) // {Madushan 27}
    }




---------- Embedded struct ----------

* Embed a struct to a struct:

      type A struct {
        ...
      }

      type B struct {
        A
        ...
      }


      * All fields and methods of A are directly available to B and we can
        override them if required.

      * To provide fields related to A, we have to use "A" as the field name
        inside B.




* Embed an interface to a struct:

      type A interface {
        ...
      }

      type B struct {
        A
        ...
      }


      * B has an anonymous field "A".

      * That field can hold any instance of a type that implements the A interface.

      * Simply we can think all methods of that instance which are defined
        inside A interface are directly copy paste to struct B.

      * But we can override that methods if required.




* Use one struct inside another.


Ex:

    type Address struct {
      city    string
      country string
    }

    type Person struct {
      name    string
      age     int
      address Address
    }

    func main() {
      p := Person{
        name: "Madushan",
        age:  26,
        address: Address{
          city:    "Kurunegala",
          country: "Sri Lanka",
        },
      }

      fmt.Println(p.address.city)    // Kurunegala
      fmt.Println(p.address.country) // Sri Lanka
    }




---------- Anonymous fields ----------

* We can access properties of a anonymous field directly without nested field
  names (field promotion).


Ex:

    type Address struct {
      city    string
      country string
    }

    type Person struct {
      name string
      age  int
      Address
    }

    func main() {
      p := Person{
        name: "Madushan",
        age:  26,
        Address: Address{
          city:    "Kurunegala",
          country: "Sri Lanka",
        },
      }

      fmt.Println(p.city)    // Kurunegala
      fmt.Println(p.country) // Sri Lanka
    }




---------- Embedded methods ----------

* We can call methods in embedded structs with anonymous fields directly without
  nested field names (method promotion).


Ex:

    type Rectangle struct {
      width  float64
      height float64
    }

    func (r Rectangle) area() float64 {
      return r.width * r.height
    }

    type Shape struct {
      Rectangle
    }

    func main() {
      r := Rectangle{
        width:  22.9,
        height: 11.8,
      }

      s := Shape{
        Rectangle: r,
      }

      fmt.Println(s.area())
    }


* In this case all fields and methods of "Rectangle" is directly accessible to
  "Shape" (If it is an interface, all method definitions are available to).

* We can think all fields and methods of "Rectangle" struct are copy paste to
  "Shape" struct.




---------- Struct comparison ----------

* Structs do not supports: <, <=, >, >=


* It supports != and == for structs in same type.


Ex:

    type Person struct {
      name string
      age  int
    }

    func main() {
      p1 := Person{
        name: "Madushan",
        age:  26,
      }

      p2 := Person{
        name: "Madushan",
        age:  26,
      }

      fmt.Println(p1 == p2) // true
    }

---------- Methods -------------------------------------------------------------

* We can define methods with both value receivers and pointer receivers.

      1. Value receiver:

          * A value receiver gets a copy of the value for following types:

              int, float, string, bool
              array, struct

          * Otherwise the value receiver is a reference to the original data.


      2. Pointer receiver:

          * A pointer receiver gets the memory address of the original data.


* Go does not support method overloading.


Ex: (With value receiver)

    type Person struct {
      firstName string
      lastName  string
      age       int
    }

    func (p Person) fullName() string {
      return p.firstName + " " + p.lastName
    }

    func main() {
      p := Person{
        firstName: "Madushan",
        lastName:  "Wijewardhana",
        age:       26,
      }
      fmt.Println(p.fullName())
    }


Ex: (With pointer receiver)

    type Person struct {
      name string
      age  int
    }

    func (p *Person) incrementAge() {
      p.age++ // same as (*p).age++
    }

    func main() {
      p := Person{
        name: "Madushan",
        age:  26,
      }

      p.incrementAge()

      fmt.Println(p) // {Madushan 27}
    }


* Always use pointer receivers unless you have a strong reason not to.


* We can define methods for any user defined types also.

      Ex:

          type MyInt int

          func (i MyInt) isPositive() bool {
            return i > 0
          }

          func main() {
            value1 := MyInt(1)
            value2 := MyInt(-1)

            fmt.Println(value1.isPositive())
            fmt.Println(value2.isPositive())
          }


* If we don't want to access the type instance inside the method, we can define
  the method without type instance.

      Ex:

          type MyInt int

          func (MyInt) welcome() string {
            return "Welcome"
          }

          func main() {
            value := MyInt(10)

            fmt.Println(value.welcome())
          }




* Go automatically takes the address or dereferences values when calling
  methods, so both values and pointers can call value or pointer receiver
  methods.

      Ex (With value receiver):

          type Person struct {
            Name string
            Age  int
          }

          func (p Person) showDetails() {
            fmt.Println("Name", p.Name)
            fmt.Println("Age: ", p.Age)
          }

          func main() {
            p := Person{
              Name: "John",
              Age:  30,
            }

            p.showDetails()
          }




      Ex (With pointer receiver):

          type Person struct {
            Name string
            Age  int
          }

          func (p *Person) showDetails() {
            fmt.Println("Name", p.Name)
            fmt.Println("Age: ", p.Age)
          }

          func main() {
            p := Person{
              Name: "John",
              Age:  30,
            }

            p.showDetails()
          }


* Go does not perform automatic address-taking or Dereferencing for interfaces.
  A type implements an interface only if its actual method set maatches exactly.

      Ex (With value receiver):

          type Person struct {
            Name string
            Age  int
          }

          func (p Person) ShowDetails() {
            fmt.Println("Name", p.Name)
            fmt.Println("Age: ", p.Age)
          }

          type Data interface {
            ShowDetails()
          }

          func tryInterface(data Data) {
            data.ShowDetails()
          }

          func main() {
            p := Person{
              Name: "John",
              Age:  30,
            }

            tryInterface(p) // Correct

            tryInterface(&p) // Correct
          }


          * The reason is a method with a value receiver is included in the
            method set of both the value type T and the pointer type *T.




      Ex (With pointer receiver):

          type Person struct {
            Name string
            Age  int
          }

          func (p *Person) ShowDetails() {
            fmt.Println("Name", p.Name)
            fmt.Println("Age: ", p.Age)
          }

          type Data interface {
            ShowDetails()
          }

          func tryInterface(data Data) {
            data.ShowDetails()
          }

          func main() {
            p := Person{
              Name: "John",
              Age:  30,
            }

            tryInterface(p) // Wrong

            tryInterface(&p) // Correct
          }


          * The reason is a method with a pointer receiver is included only in
            the method set of *T, not T.




---------- (Defining Methods For Custom Types) ----------

* We can define a custom type using any existing type.


* When we convert a compatible instance to that custom type, all defined methods
  for that custom type are available to the convered instance.


* Ex 1: Defining custom struct type:

      type Person struct {
        Name string
        Age  int
      }

      func (p *Person) showInfo() {
        fmt.Println("Name:", p.Name)
        fmt.Println("Age :", p.Age)
      }

      func main() {
        p := struct {
          Name string
          Age  int
        }{
          Name: "John",
          Age:  32,
        }

        var person Person = p

        person.showInfo()
      }


* Ex 2: Defining custom function type:

      type MyFunc func(int) int

      func (f MyFunc) Call(x int) int {
        return f(x)
      }

      func square(x int) int {
        return x * x
      }

      func main() {
        var f MyFunc = square
        value := f.Call(5)

        fmt.Println(value)
      }

---------- Interfaces ----------------------------------------------------------

* Interfaces provide a way to specify behaviors. They defines a set of method
  signatures that a type must implements.


* We cannot define methods for interfaces.


* The default value of an interface is "nil".


Ex:

    type Geometry interface {
      area() float64
      perim() float64
    }

    // ---------------------------------------

    type Rectangle struct {
      width, height float64
    }

    func (r Rectangle) area() float64 {
      return r.width * r.height
    }

    func (r Rectangle) perim() float64 {
      return 2 * (r.width + r.height)
    }

    // ---------------------------------------

    type Circle struct {
      radius float64
    }

    func (c Circle) area() float64 {
      return math.Pi * c.radius * c.radius
    }

    func (c Circle) perim() float64 {
      return 2 * math.Pi * c.radius
    }

    // ---------------------------------------

    func measure(g Geometry) {
      fmt.Println("Area :", g.area())
      fmt.Println("Perim:", g.perim())
    }

    // ---------------------------------------

    func main() {
      r := Rectangle{
        width:  10.1,
        height: 20.8,
      }

      c := Circle{
        radius: 12.4,
      }

      measure(r)
      measure(c)
    }

---------- Generics ------------------------------------------------------------

* [T any] or [T type1 | type2 | ...]


* To get the default value of a generic type we can use following approach:

      var zero T




---------- (With functions) ----------

    func swap[T any](a, b T) (T, T) {
      return b, a
    }

    func main() {
      x1, y1 := 1, 2
      x1, y1 = swap(x1, y1)
      fmt.Println(x1, y1) // 2 1

      x2, y2 := "John", "Doe"
      x2, y2 = swap(x2, y2)
      fmt.Println(x2, y2) // Doe John
    }




---------- (With structs) ----------

* Generics in struct definition:

    type Stack[T any] struct {
      elements []T
    }


* Generics in struct methods:

      func (s *Stack[T]) push(element T) {
        s.elements = append(s.elements, element)
      }

      func (s *Stack[T]) pop() (T, bool) {
        if s.isEmpty() {
          var zero T
          return zero, false
        }

        element := s.elements[len(s.elements)-1]
        s.elements = s.elements[:len(s.elements)-1]
        return element, true
      }

      func (s *Stack[T]) isEmpty() bool {
        return len(s.elements) == 0
      }

---------- Errors --------------------------------------------------------------

* In Go errors are also values.


Ex:

    func sqrt(value float64) (float64, error) {
      if value < 0 {
        return 0, errors.New("square root of negative number")
      }

      return math.Sqrt(value), nil
    }

    func main() {
      value, err := sqrt(-1)
      if err != nil {
        fmt.Println(err)
        return
      }
      fmt.Println(value)
    }


* To create error value with a static string:

      errors.New(<string>)


* To create error value with a dynamic string:

      fmt.Errorf(<string>)




---------- (Custom errors) ----------

* In go we use built in "error" interface as the type of any error.

      type error interface {
        Error() string
      }


* To make our custom error struct compatible with error interface, we have to
  define a method named "Error()" with a pointer receiver.


* When return the custom error, return its memory address (pointer).


Ex:

    type AppError struct {
      code    int
      message string
    }

    func (e *AppError) Error() string {
      return fmt.Sprintf("Error %d: %s\n", e.code, e.message)
    }

    func sqrt(value float64) (float64, error) {
      if value < 0 {
        return 0, &AppError{
          code:    404,
          message: "sample error",
        }
      }

      return math.Sqrt(value), nil
    }

    func main() {
      value, err := sqrt(1)
      if err != nil {
        fmt.Println(err)
        return
      }

      fmt.Println(value)
    }

---------- Regular expressions -------------------------------------------------

* To represent exactly one character use it directly. But if it is one of below,
  then use it with \ (backslash)

      *
      .
      +
      ?
      ^
      $
      |
      - (dash)
      \ and /
      ( and )
      [ and ]
      { and }

      (16 characters)


* We use [] to represent exactly one character and that character has 2 or more
  posibilities.

      .           - any character (dot)
      [abc]       - "a" or "b" or "c"
      [^abc]      - anything except "a" or "b" or "c"
      [a-z]       - any lowercase english alphabet
      [A-Z]       - any uppercase english alphabet
      [0-9]       - any number
      [a-zA-Z0-9] - any lowercase or uppercase alphabet or any number.


* We use quantifiers to define multiple occurances for a character:

      []?     - 0 or 1 times
      []+     - 1 or more times
      []*     - 0 or more times
      []{n}   - exactly n times
      []{n,}  - n or more times
      []{n,m} - n or more times, but less than m times


* We use () to group multiple characters.


* To create the expression always use ``:

      re := regexp.MustCompile(`<regex>`)


* Check a given string is matched to the regex:

      re.MatchString(<string>)


* Extract all matches from a given string for a given regex:

      re.FindAllString(<strings>, -1)


* Replace all sub strings that are matched with given regex with given sub
  string.

      re.ReplaceAllString(<source>, <subString>)

---------- Time ----------------------------------------------------------------

* Current local time:

      time.Now()


* Specify a exact date:

      time.Date(
        <year>,        - like 2025
        <month>,       - time.<month> or 1 - 12
        <day>,         - from 1 to 31
        <hour>,        - from 0 to 23
        <minutes>,     - from 0 to 59
        <seconds>,     - from 0 to 59
        <nanoseconds>, - from 0 to 999999999
        <timezone>     - time.UTC / time.Local
      )


* Convert local time to UTC time:

      localTime.UTC()


* Convert time to local:

      anyTime.Local()


* Go uses a reference time instead of format codes like other languages:

      Year     : 2006 (4 digits) / 06 (2 digits)
      Month    : 1 / 01 / Jan / January
      Day      : Mon / Monday
      Date     : 2 / 02
      Hour     : 3 / 03 (12h) | 15 (24h)
      minute   : 4 / 04
      second   : 5 / 05
      timezone : -0700


* Convert a string to a time:

      time.Parse(<layout>, <timeString>)


      Ex:
          date, err := time.Parse("2006-01-02 15:00:00 -0700", "2009-11-03 09:00:00 +0530")


      Note:

          Layout is the format of the time string which uses pre-defined
          references.

          If layout contains timezone (-0700), then given time will be considered
          in provided timezone. Otherwise provided time will be considered in
          UTC.


* Convert a time to specific string format:

      <time>.Format(<layout>)

      Note:
          Layout for ISO 8601 standard string: RFC3339


* To get weekday of a given time:

      <time>.Weekday() // like Sunday, Monday, ...


* Get a given time in specific timezone:

      loc, _ := time.LoadLocation("Asia/Colombo")
      <time>.In(loc)


* Round a time:

      now.Round(time.Hour) - To nearest hour

      now.Truncate(time.Hour):
          Same as Round, but rounds to floor only.


* Convert a given time to unix:

      <time>.Unix()


* Convert a unix time to time.Time:

      time.Unix(<time>, 0)


* Convert a time to iso string:

      time.Now().Format(time.RFC3339)




---------- (Time Calculations) ----------

* Add a given duration to a time:

      <time>.Add(<duration>)


* Subtract a given duration from a time:

      <time>.Add(-1 * <duration>)


* Get the duration between two given times:

      <time1>.Sub(<time2>)

      * If time2 is later than time1, you get a negative duration.


* Divide a duration by a given integer:

      * If the number is a constant:

            interval := <duration> / <int_value>


      * If the number is a variable:

            interval := <duration> / time.Duration(<int_value>)


* Multiply a duration by a given integer:

      * If the number is a constant:

            duration := <duration> * <number>


      * If the number is a variable:

            duration := <duration> * time.Duration(<number>)


      * If the speed is 5m per second and the duration is given:

            speed := 5 // meters per second
            duration := <given_duration>

            * If the speed is a float:

                length := speed * duration.Seconds()


            * If the speed is an int:

                length := float64(speed) * duration.Seconds()


* To convert a string duration to time.Duration:

      h  -> hours
      m  -> minutes
      s  -> seconds
      ms -> milliseconds


      * Allowed formats: 30h, 30ms, 1h30m, ... (spaces are not allowed)


      * Conversion:

            duration, err := time.ParseDuration("")

---------- URL parsing ---------------------------------------------------------

func main() {
	urlStr := "https://username:password@www.example.com:8080/segment1/segment2?query1=param1&&query2=param2#fragment"

	parsedUrl, err := url.Parse(urlStr)

	fmt.Println(err)

	fmt.Println(parsedUrl.Scheme) // https

	fmt.Println(parsedUrl.User)            // username:password
	fmt.Println(parsedUrl.User.Username()) // username
	fmt.Println(parsedUrl.User.Password()) // password true

	fmt.Println(parsedUrl.Host)       // www.example.com:8080
	fmt.Println(parsedUrl.Hostname()) // www.example.com
	fmt.Println(parsedUrl.Port())     // 8080

	fmt.Println(parsedUrl.Path) // /segment1/segment2

	fmt.Println(parsedUrl.Query())  // map[query1:[param1] query2:[param2]]
	fmt.Println(parsedUrl.RawQuery) // query1=param1&&query2=param2

	fmt.Println(parsedUrl.Fragment) // fragment
}

---------- bufio package -------------------------------------------------------

* Wrapper around input/output (like files, network connections, or standard
  input/output) to make reading and writing more efficient.


* How reading works:

      Beffering reads a fixed size-chuck (like 4kb, 8kb , or 16kb) from the file
      into memory. Not the whole file.

      Then your program reads from that in-memory chunk instead of calling the
      os each time.

      When buffer runs out, bufio automarically fills it again with the next
      chunk.


* How writing works:

      When you perform writing with buffer, it doesn't write data to disk
      or network immediately.

      Stores the data in a memory buffer (temporary area).

      Writes the entire buffer to the underlying writer only when:

          - The buffer is full

          - You explicitly flush it.




---------- (Reading) ----------

* Initialize a reader:

      reader := bufio.NewReader(<io_reader>)


* Read data

      buff := make([]byte, <k>)
      n, err := reader.Read(buff)


      k - Number of bytes to read at once. if k <= 4096, reader.Read reads
          k number of bytes from the file. If k > 4096, then reader.Read reads
          4096 bytes from the file.

      n - Number of bytes read by reader.Read


* reader.ReadString(<delimiter>):

      Read data from the reader until a specified delimiter and returns it as a
      string.




---------- (Writing) ----------

    func main() {
      writer := bufio.NewWriter(os.Stdout)

      buff := []byte("Hello, bufio package!\n")
      n, err := writer.Write(buff)
      if err != nil {
        fmt.Println(err)
        return
      }
      writer.Flush()
      fmt.Println(n)
    }


* writer.WriteString(<str>):

      To write strings.

---------- Base64 coding -------------------------------------------------------

* Represent any data using a string.


* How it works:

      * Step 1: Group data into 24-bit chunks.

      * Step 2: Split each 24-bit chunk into 4 6-bit groups.


* Base64 fixed character table:

      Index 0-25  -> A-Z
      Index 26-51 -> a-z
      Index 52-61 -> 0-9
      Index 62    -> +
      Index 63    -> /


* Base64 encoded string size is ~33% bigger than original size of the data.

      * Suppose we have 3 bytes of data

      * Base64 works like this:

            * Takes 3 bytes (24 bits) of input.
            * Splits into 4 groups of 6 bits.
            * Each 6-bit value maps to a printable ASCII character.

      * Original: 3 bytes = 24 bits.

      * Base64 output: 4 characters = 4 bytes = 32 bits.

      * 32/24 = 1.33 = ~33% larger.


Ex:
    * Convert "hello" to base64:

    func main() {
      msg := "hello"

      byteSlice := []byte(msg) // [104 101 108 108 111]

      encoded := base64.StdEncoding.EncodeToString(byteSlice) // aGVsbG8=
    }

    * When we convert "hello" to a byte slice, it becomes [104 101 108 108 111].

    * ASCII table:

        Char Decimal Binary
        h    104     0110 1000
        e    101     0110 0101
        l    108     0110 1100
        o    111     0110 1111

    * In base64 we divide bits in to 6 bit groups:

          -> 104 101 108 108 111
          -> 01101000 01100101 01101100 01101100 01101111
          -> 01101000 01100101 01101100 | 01101100 01101111 (split into 24-bit chunks)
          -> 011010 000110 010101 101100 | 011011 000110 1111 (split each group into 6-bit chunks)
          -> 011010 000110 010101 101100 | 011011 000110 111100 000000 (replace missing bits with zeros)
          -> 26 6 21 44 | 27 6 60 0
          -> a G V s | b G 8 A
          -> a G V s | b G 8 =

    * Why replace A with =:

          * For last group we calculate the answer for following equation:

                original_data_length_in_bytes % 3

          * If the answer is 1, we replace last 2 characters with ==

          * If the answer is 2, we replace last characters with =




---------- (Encoding) ----------

---------- (Decoding) ----------

---------- (URL encoding / decoding) ----------

* Difference:

      1. + replaced with - (dash)
      2. / replaced with _ (underscore)


* To encode and decode strings to pass via urls. Because urls consider some
  base64 characters as url specific characters. (like /, +)


* base64.URLEncoding.EncodeToString(<byte_slice>)


* base64.URLEncoding.DecodeString(<string>)




---------- (Raw Encoding) ----------

* Raw means remove =

* RawStdEncoding: ignore =

* RawURLEncoding: ignore =




* Number of characters when we convert a byte slice to a base64 string:

      * Consider 32 byte slice

      Step 1: Divide number of bytes in 3 and get the ceil value and rest of it.

            = 32 / 3
            = 10.66
            = ceil: 11, rest 2

      Step 2: Multiply that value by 4. This is the length of the base64 string.

            length of the string: 4 * 11 = 44

      Step 3: Last (3 - rest) characters are =

---------- IO package ----------------------------------------------------------

---------- (io.Reader) ----------

* A reader is any type that implements the "io.Reader" interface.


* The "io.Reader" interface has only one method signature called "Read":

      type Reader interface {
        Read(p []byte) (n int, err error)
      }


* "io.Reader" represent a source you can read bytes from.


* If a type has a "Read" method with this exact signature, then it is a
  reader.


* Readers allow you to read data abstractly, without caring where the data
  comes from.


* Even if "Read" returns an error, the "n" value may still be greater than
  0, meaning some data was successfully read before the error happened. So
  your code must use the data (n bytes) first, and only then handle error.


* Any proper "io.Reader" implementation must return "io.EOF" error when it
  reaches the end of its data.




* Some common implementations of the "io.Reader":

      os.File:
          Used to read data from a file.


      strings.Reader:
          Creates a new reader from the provided string. But we cannot create a
          string reader directly using "strings.Reader". The reason is it
          requires internal fields to be initialized correctly.

          Therefore we use "strings.NewReader" constructor function to create an
          initialized string reader directly.


      bufio.Reader:
          Implements "Read" for improved performance, using buffers for reading.

          But instead of manually create and initialize "bufio.Reader" we
          directly use "bufio.NewReader" constructor function.


      http.Request.Body:
          Allows you to supply the request body via reader.


      bytes.Reader:
          Allows you to read from a byte slice.

          But instead of manually create and initialize "bytes.Reader" we use
          "bytes.NewReader" constructor function.


      bytes.Buffer:
          Used for reading from in-memory byte buffers.

          In this case often we use "bytes.Buffer" in practice intead of using
          "bytes.NewBuffer" constructor function.




* Read data from a reader

    Read in chunks with a buffer:

        func readFromReader(r io.Reader) error {
          buff := make([]byte, 4)

          for {
            n, err := r.Read(buff)

            // Check number of read bytes before error
            if n > 0 {
              fmt.Print(string(buff[:n]))
            }

            // Check EOF before other errors
            if err == io.EOF {
              return nil
            }

            if err != nil {
              return err
            }
          }
        }


    To read in chunks with a inbuilt function, use following method. It uses
    a byte slice with length 512 initially:

        data, err := io.ReadAll(reader)




* Combine multiple readers to one reader:

      r1 := strings.NewReader("hello")
      r2 := strings.NewReader("world")

      r := io.MultiReader(r1, r2)


      process:

          It reads from the first reader until it hits io.EOF

          Then it automatically moves to next reader. then next reader, and so on.

          It returns io.EOF only when all readers are finished.

---------- (io.Pipe) ----------

* Creates an in-memory synchronous pipe: a connected io.Reader and io.Writer
  pair. Whatever you write to the writer can be read from the Reader.

      pr, pw := io.Pipe()

          pr: *io.PipeReader (implements io.Reader)
          pw: *io.PipeWriter (implements io.Writer)

          They are connected internally. Data flows from writer -> Reader.


* Key characteristics:

      1. Synchronous:

          A PipeWriter.Write() call blocks until the data is read.

          A PipeReader.Read() call blocks until the data is written.

          So both sides must run in different goroutines to avoid deadlock.


      2. No buffering:

          There is no internal buffer.. It only transfers data when both sides
          are ready. Think of it like streaming data, not storing it.


      3. Common for streaming between goroutines:

          Useful for:

              Streaming large data without allocating memory

              Connecting encoders/decoders

              Passing data to compressors, HTTP requests, etc


* When to use io.Pipe():

      You want to stream data between goroutines.

      You don't want to store entire data in memory.

      You need an io.Reader but your data is produced dynamically.

      You need an io.Writer but the consumer expects an io.Reader.


* Closing PipeWriter:

      pw.Close():

          Signals EOF to the reader.

          The reader (pr.Read()) will return io.EOF once all written data is
          consumed.


      pw.CloseWithError(err):

          Signals an error to the reader instead of EOF.

          The reader's Read() returns (n, err) with your custom error.


      Most common usage:
          The writer finishes or fails -> tell the reader why


* Closing PipeReader:

      pr.Close():

          Causes ongoing/next writes on pw.Write() to return ErrClosedPipe.

          It means "The reader is gone. Stop writing".


      pr.CloseWithError(err):

          Writer (pw.Write()) immediately fails with your custom error instead
          of ErrClosedPipe.


      Used when:
          Reader aborts early and wants to stop the writer.


* Writer closes pw to signal EOF or error to reader.

* Reader closes pr to signal cancellation or error to writer.


Ex:

    func pipeExample() {
      pr, pw := io.Pipe()

      go func() {
        pw.Write([]byte("hello pipe"))
        pw.Close()
      }()

      var buff bytes.Buffer
      buff.ReadFrom(pr)

      fmt.Println(buff.String()) // hello pipe
    }

    func main() {
      pipeExample()
    }


Ex: Case 1:

    Since no data has been written yet, Read will block and wait until:

        1. pw.Write() writes some data

        or

        2. pw.Close() / pw.CloseWithError() is called. Then Read returns 0 with
           error.


            func pipeExample() {
              pr, pw := io.Pipe()

              go func() {
                time.Sleep(time.Second * 5)
                pw.Write([]byte("sample data"))

              n, err := pr.Read(buff)
              if err != nil {
                log.Fatalln("Failed to read data:", err)
              }

              fmt.Println(string(buff[:n]))
            }

            func main() {
              pipeExample()
            }




Ex: Case 2:

    Write will also block until a Read is available to receive the data. This is

---------- (io.Copy) ----------

* It reads from a source (io.Reader) and writes to a destination (io.Writer)
  until EOF, using an internal 32kb buffer.


* Stops when:

      1. src returns EOF, or

      2. A read/write error occurs


Ex 1:

    func main() {
      src, err := os.Open("src.txt")
      if err != nil {
        log.Fatalln("Failed to open the file:", err)
      }
      defer src.Close()

      dst, err := os.Create("destination.txt")
      if err != nil {
        log.Fatalln("Failed to create the file:", err)
      }
      defer dst.Close()

      n, err := io.Copy(dst, src)
      if err != nil {
        log.Fatalln("Failed to copy data:", err)
      }

      fmt.Println("Successfully copied", n, "bytes")
    }


Ex 2:

    func main() {
      pr, pw := io.Pipe()

      go func() {
        time.Sleep(time.Second * 5)

        defer pw.Close()
        pw.Write([]byte("sample data"))
      }()

      io.Copy(os.Stdout, pr)
    }




* io.CopyBuffer:

      When you want to provide your own buffer instead of Go's default 32kb
      buffer.

          buff := make([]byte, 1024)
          io.CopyBuffer(dst, src, buff)


* io.CopyN:

      Copy exactly N bytes.

          io.CopyN(dst, src, 10) // copy only first 10 bytes

---------- (Goroutines) --------------------------------------------------------

* Goroutines are just functions that leave the main thread and run in the
  background and come back to join the main thread once the functions are
  finished/ready to return any value.


* "main" function is also a goroutine. It is the main goroutine of a go program.


* Goroutines are non-blocking.


* Simplify concurrent programming.


* Efficiently handle parallel tasks such as I/O operations, calculations, and
  more.


* Provide a way to perform tasks concurrently without manually managing threads.


* Managed by the Go Runtime Scheduler. Uses M:N Scheduling Model.


* Syntax:

      go func (<args>) {
        // body
      } ()

      or

      go <functionName>(<args>)


Ex:

    func sayHello() {
      time.Sleep(time.Second * 1)
      fmt.Println("Hello")
    }

    func main() {
      go sayHello()
    }


  * In this example we cannot see anything is printed to the console.

  * The reason is we run the "sayHello" function as a goroutine. It waits 1
    second before printing the "Hello". But main function ends before printing.
    If the main function is ended, our program is also ended.




---------- (Pass values to goroutines) ----------

* If the value is an int, float, string, bool, array, struct, and if you use the
  value directly inside a goroutine, a new copy won't be created.

Ex:

    func main() {
      count := 1

      go func() {
        time.Sleep(2 * time.Second)

        fmt.Println("Count:", count) // Count: 2
      }()

      count++

      time.Sleep(3 * time.Second)
    }




* If pass as a parameter, a new copy will be created:

Ex:

    func main() {
      count := 1

      go func(count int) {
        time.Sleep(2 * time.Second)

        fmt.Println("Count:", count) // Count: 1
      }(count)

      count++

      time.Sleep(3 * time.Second)
    }




* If the value is another data type, then a new copy won't be created even if
  you pass it directly or as a parameter.

---------- Channels ------------------------------------------------------------

* Enable safe and efficient communication between concurrent goroutines. Helps
  to synchronize and manage the flow of data in concurrent programs.


* There are 2 types of channels:

      1. Unbuffered channels
      2. Buffered channels


* Empty structs (struct{}{}) are often used in channels for signaling because
  they are perfect for "event happened" notifications, not for passing data. An
  empty struct has no fields, so it occupies 0 bytes in memory.

      ch := make(chan struct{}) // Creation

      ch <- struct{}{} // Send signal

      <-ch // Access signal

--------------------------------------------------------------------------------




---------- Unbuffered channels -------------------------------------------------

* Don't have an associated storage.


* Need an immediate receiver.


* Creation:

      channelName := make(chan <type>)


* Send a value to an unbuffered channel:

      channelName <- value

      * Sending a value to an unbuffered channel is blocking by default.

      * The sending goroutine pauses until another goroutine is ready to
        receive.


* Receive a value from an unbuffered channel:

      value := <-channelName

      * Receiving from an unbuffered channel is also blocking by default.

      * The receiving goroutine waits until someone sends a value into the
        channel.


Ex 1: Waiting for the sender:

    func main() {
      numberChannel := make(chan int)

      go func() {
        time.Sleep(time.Second * 2)

        numberChannel <- 10
      }()

      fmt.Println("Before receiver")
      number := <-numberChannel
      fmt.Println("After receiver")

      fmt.Println(number)
    }


Ex 2: Waiting for the receiver:

    func main() {
      numberChannel := make(chan int)

      go func() {
        fmt.Println("Before sender")
        numberChannel <- 10
        fmt.Println("After sender")
      }()

      time.Sleep(time.Second * 2)

      number := <-numberChannel

      fmt.Println(number)

      time.Sleep(time.Second * 1)
    }

--------------------------------------------------------------------------------




---------- Buffered channels ---------------------------------------------------

* A channel with an associated storage.


* Allows a channel to hold a limited number of values before blocking
  the sender.


* A buffered channel allows asynchronous communication. It allows senders to
  continue working without blocking until the buffer is full.


* A buffered channel will only block when the buffer is full.


* Creation:

      channelName := make(chan <type>, <capacity>)


* Send a value to an buffered channel:

      channelName <- value

      * This is blocking only if the buffer is full.

      Ex:

          func main() {
            numberChannel := make(chan int, 2)

            numberChannel <- 10
            numberChannel <- 20

            go func() {
              time.Sleep(time.Second * 2)

              <-numberChannel
            }()

            fmt.Println("Blocking start")
            numberChannel <- 30
            fmt.Println("Blocking end")
          }


* Receive a value from a buffered channel:

      value := <-channelName

      * This is blocking only if the buffer is empty.

      Ex:

          func main() {
            numberChannel := make(chan int, 2)

            go func() {
              time.Sleep(time.Second * 2)

              numberChannel <- 10
            }()

            fmt.Println("Blocking start")
            number := <-numberChannel
            fmt.Println("Blocking end")

            fmt.Println(number)
          }

--------------------------------------------------------------------------------




---------- Channel synchronization ---------------------------------------------

* Ensures that data is properly exchanged between Goroutines.


* Coordinates the execution flow to avoid race conditions and ensure predictable
  behavior.


* Helps manage the lifecycle of Goroutines and the completion of tasks.


Ex: A basic example:

    func main() {
      done := make(chan int)

      go func() {
        fmt.Println("Working...")
        time.Sleep(time.Second * 2)

        done <- 10
      }()

      <-done

      fmt.Println("Finished")
    }


Ex: Listening to an unbuffered channel using a for loop:

    func main() {
      ch := make(chan int)

      go func() {
        for index := range 5 {
          time.Sleep(time.Second)

          ch <- index
        }
        close(ch)
      }()

      for value := range ch {
        fmt.Println("Received value:", value)
      }

      fmt.Println("End of the program")
    }

    * At the end of the sending values, we have to close the channel to notify
      the for loop:

          close(ch)

--------------------------------------------------------------------------------




---------- Channel directions --------------------------------------------------

* Improve code clarity and maintainability.


* Prevent unintended operations on channels.


* Enhance type safety by clearly defining the channel's purpose.




---------- (Bidirectional channels) ----------

* By default a channel is bidirectional.


Ex:

    channel := make(chan <type>)

    Type: chan <type>


* You can send and receive:

      Send   : channel <- value
      Receive: value := <-channel




---------- (Write-only channels) ----------

* Can only send. Cannot receive.


Ex:

    func producer(ch chan<- int) {
      ch <- 10 // Allowed

      <-ch // Compile-time error
    }

    Type: chan<- <type>




---------- (Read-only channels) ----------

* Can only receive. Cannot send.


Ex:

    func consumer(ch <-chan int) {
      <-ch // Allowed

      ch <- 10 // Compile-time error
    }




---------- (Conversions) ----------

* You can restrict, but not widen, channel capabilities.


Ex 1:

    func main() {
      var ch chan int

      var sendOnly chan<- int = ch // Allowed
      var receiveOnly <-chan int = ch // Allowed
    }


Ex 2:

    func main() {
      var ch chan<- int

      var both chan int = ch // Compile-time error
    }


Ex 3: A typical real-world pattern:

    func generate(nums ...int) <-chan int {
      out := make(chan int)

      go func() {
        defer close(out)

        for _, v := range nums {
          time.Sleep(time.Second)

          out <- v
        }
      }()

      return out
    }

    func main() {
      out := generate(10, 20, 30, 40, 50)

      for v := range out {
        fmt.Println(v)
      }
    }

    * Caller can only receive, ensuring correct usage.

---------- Multiplexing using Select -------------------------------------------

* You can think of "select" as the channel version of "switch".


* Allowed only inside a function.


* What "select" does?

      * Send or Receive on multiple channels.

      * Proceed with whichever operation is ready first.

      * Optionally handle timeout or non-blocking behavior.


* Important rules:

      * "select" works only with channels

      * All channel operations must be send or receive

      * Default is optional

      * Closing a channel makes receive cases immediately ready


* Basic syntax:

    select {
    case v := <-ch1:
      // receive from channel-1

    case ch2 <- x:
      // send to channel-2

    default:
      // optional: runs if no case is ready
    }




* Blocking behavior:

      * If no case is ready, "select" blocks.

      * If multiple cases are ready, Go picks one randomly (fairness)




* Non-blocking select (using default):

      If you add the default case to the select, it never blocks.


      * Non blocking receive operation:

            func main() {
              ch := make(chan int)

              select {
              case v := <-ch:
                fmt.Println("Received:", v)

              default:
                fmt.Println("No message available")
              }
            }


      * Non blocking send operation:

            func main() {
              ch := make(chan int)

              select {
              case ch <- 1:
                fmt.Println("Message is sent")

              default:
                fmt.Println("Channel is not ready to receive")
              }
            }


      * Non blocking operation in real-time systems:

            func main() {
              data := make(chan int)
              quit := make(chan bool)

              go func() {
                for {
                  select {
                  case d := <-data:
                    fmt.Println("Data received:", d)

                  case <-quit:
                    fmt.Println("Stopped")
                    return

                  default:
                    fmt.Println("Waiting for data...")
                    time.Sleep(500 * time.Millisecond)
                  }
                }
              }()

              for i := range 5 {
                data <- i
                time.Sleep(time.Second)
              }
              quit <- true
            }




* Timeout pattern:

    func main() {
      ch := make(chan int)

      go func() {
        time.Sleep(time.Second * 5)

        ch <- 10
      }()

      select {
      case v := <-ch:
        fmt.Println("received:", v)

      case <-time.After(time.Second * 2):
        fmt.Println("timeout")
      }
    }

    * If no value arrives within 2 seconds, timeout triggers.




* Infinite loop with select (event loop):

      for {
        select {
        case job := <-jobs:
          process(job)

        case <- time.After(time.Second * 2):
          fmt.Println("timeout")
        }
      }




---------- (Receiving on a closed channel in select) ----------

* It is always recommended to close channels only from the sender side.


* If a channel in a select is closed:

      * Its receive case becomes always ready.

      * Immediately returns the zero value of the channel type.

      * ok == false


* Ex:

    func main() {
      ch := make(chan int)

      go func() {
        time.Sleep(time.Second * 1)

        close(ch)
      }()

      select {
      case v, ok := <-ch:
        if ok {
          fmt.Println("received:", v)
        } else {
          fmt.Println("closed:", v) // closed: 0
        }

      case <-time.After(time.Second * 2):
        fmt.Println("timeout")
      }
    }


* Closed channel + select + for loop:

      for {
        select {
        case v := <-ch:
          fmt.Println(v)
        }
      }


    * If ch is closed:

          * select keeps choosing that case.

          * You get infinite zero values.


    * Correct version:

          for {
            select {
            case v, ok := <-ch:
              if !ok {
                return
              }
              fmt.Println(v)
            }
          }




---------- (Sending on a closed channel in select) ----------

* Unlike receive, send on a closed channel always panics, even inside select.

--------------------------------------------------------------------------------




---------- Closing channels ----------------------------------------------------

* We close channels to signal completion. It indicates that no more data will be
  sent on the channel.


* To close a channel we use close function:

      close(ch)


* After a channel is closed:

      * No more values can be sent to that channel.

      * No more values can be received to that channel if it is a unbuffered
        channel.

      * We can receive values from a closed channel if it is a buffered channel.
        Because it may have some values stored in the buffer.


* Best practices for closing channels:

      1. Close channels only from the sender.

      2. Avoid closing channels more than once.

      3. Avoid closing channels from multiple goroutines.




* Receiving from a closed channel:

      func main() {
        ch := make(chan int)

        close(ch)

        v, ok := <-ch

        if !ok {
          fmt.Println("Channel is closed")
          return
        }

        fmt.Println("Received:", v)
      }




* Range over a closed channel:

      func main() {
        ch := make(chan int)

        go func() {
          for i := range 5 {
            ch <- i

            time.Sleep(time.Second)
          }

          close(ch)
        }()

        for v := range ch {
          fmt.Println("Received:", v)
        }
      }




* Common patterns for closing channels:

      1. Pipeline Pattern
      2. Worker Pool Pattern


      * Worker pool pattern will be discussed later.


      * Pipeline pattern:

            func producer(ch chan<- int) {
              for i := range 5 {
                ch <- i
              }

              close(ch)
            }

            func filter(in <-chan int, out chan<- int) {
              for v := range in {
                if v%2 == 0 {
                  out <- v
                }
              }

              close(out)
            }

            func main() {
              ch1 := make(chan int)
              ch2 := make(chan int)

              go producer(ch1)
              go filter(ch1, ch2)

              for v := range ch2 {
                fmt.Println("Received:", v)
              }
            }

---------- Channels Short Note -------------------------------------------------

* The zero value of a channel is: nil


* Create a channel:

      ch := make(chan <type>) // Unbuffered

      ch := make(chan <type>, <size>) // Buffered


* To send data to a channel always use:

      ch <- value


* To receive data from a channel always use:

      v, ok := <-ch


* We only create bidirectional channels, then pass them to goroutines as
  read-only or write-only to enforce correct usage and avoid bugs.

      Read only channel : var ch <-chan <type>

      Write only channel: var ch chan<- <type>


* Always close the channel from the sender side.


* To close a channel always use:

      close(<bidirectional_channel>) or close(<send_only_channel>)




---------- (Unbuffered channels) ----------

* Creation:

      ch := make(chan <type>)


* Sending data to a channel is blocking until receiver is ready:

      ch <- value


* Receiving data from a channel is blocking until sender sends:

      v, ok := <-ch


* Sending data to a closed channel causes a runtime panic.


* Receiving data from a closed channel:

      v, ok := <-ch

      v -----> Zero value of the type
      ok ----> false


* To receive values continuously until the channel is closed:

      for v := range ch {
        // body
      }

      * For loop breaks automatically when ok is false.
        (closed channel)




---------- (Buffered channels) ----------

* Creation:

      ch := make(chan <type>, <size>)


* Sending data to a channel is blocking only if the buffer is full:

      ch <- value


* Receiving data from a channel is blocking only if the buffer is empty:

      v, ok := <-ch


* Sending data to a closed channel causes a runtime panic.


* Receiving data from a closed channel:

      * Channel is closed, but buffer contains some values:

          v, ok := <-ch

          v -----> value
          ok ----> true


      * Channel is closed and buffer is empty:

          v, ok := <-ch

          v -----> zero value of the type
          ok ----> false


* To receive values continuously until the channel is closed and buffer is empty:

      for v := range ch {
        fmt.Println(v)
      }

      * For loop breaks automatically when ok is false.
        (closed channel + empty buffer)

---------- Context -------------------------------------------------------------

* The context package is used to carry request-scoped data, deadlines, timeouts,
  and cancellation signals across API boundaries and between goroutines.


* It is a core package and is heavily used in servers, APIs, databases, and
  concurrent programs.


* A context can carry:

      1. Cancellation signal
      2. Deadline / Timeout
      3. Key-value data (request-scoped)




* Why context exists:

      1. Handle requests (HTTP, gRPC, DB queries).
      2. Spawn multiple goroutines per request.
      3. Need to cancel work when a request is no longer needed.
      4. Need to enforce timeouts.
      5. Need to pass metadata (request ID, auth info)

      * Without context, this becomes messy and unsafe.




* Common use cases:

      1. Cancel goroutines:

          If a client disconnects, stop all work related to that request.


      2. Timeouts:

          Automatically stop operations if they take too long.


      3. Propagate request data:

          Pass request IDs, user info, tracing data safely.




* Creating base contexts:

      1. context.Background():

          * Root context (main, servers)

          * Child contexts:

                1. WithCancel
                2. WithTimeout
                3. WithDeadline
                4. WithValue


      2. context.TODO():

          * Placeholder when unsure.

          * Not special internally. It behaves the same as Background().

          * Its value is semantic, not technical.




---------- (context.Context interface) ----------

type Context interface {
  Deadline() (deadline time.Time, ok bool)
  Done() <-chan struct{}
  Err() error
  Value(key any) any
}




* ctx.Deadline():

      d, ok := ctx.Deadline():

          d  : The exact time when the context will be canceled.
          ok : true if a deadline exists, false if not.


          A context has a deadline if it was created using:

              context.WithDeadline
              context.WithTimeout




* ctx.Done():

      <-ctx.Done()

          It returns a read-only channel and we use it to listen to the
          calcellation singal.


          Mostly used with "select".


          Done() is triggered when:

              1. cancel() is called (from context.WithCancel)

                    func main() {
                      bgCtx := context.Background()

                      ctx, cancel := context.WithCancel(bgCtx)

                      go func() {
                        time.Sleep(5 * time.Second)
                        cancel()
                      }()

                      <-ctx.Done()

                      println("End")
                    }


              2. The parent context is canceled:

                    func main() {
                      bgCtx := context.Background()

                      parentCtx, parentCancel := context.WithCancel(bgCtx)

                      ctx, _ := context.WithCancel(parentCtx)

                      go func() {
                        time.Sleep(5 * time.Second)
                        parentCancel()
                      }()

                      <-ctx.Done()

                      println("End")
                    }


              3. The deadline expires:

                    context.WithDeadline
                    content.WithTimeout




* ctx.Err():

      err := ctx.Err()

          * Tells you why a context was canceled or expired.


          * Returns:

                nil                      : If context still active

                context.Canceled         : If Context was canceled manually

                context.DeadlineExceeded : If context deadline exceeded


          Ex:
              func main() {
                ctx, cancel := context.WithCancel(context.Background())

                go func() {
                  time.Sleep(5 * time.Second)

                  cancel()
                }()

                for {
                  select {
                  case <-ctx.Done():
                    fmt.Println("Stopped | Error:", ctx.Err())
                    return

                  default:
                    fmt.Println("Working... | Error:", ctx.Err())
                  }

                  time.Sleep(time.Second)
                }
              }




* ctx.Value(<key>):

      * v, ok := ctx.Value(<key>).(<type>)


      * Returns the value associated with that key, or nil if not found.


      Ex:

          type userIDKeyType string

          func main() {
            var userIDKey userIDKeyType = "userId"
            ctx := context.WithValue(context.Background(), userIDKey, 1)

            userID, ok := ctx.Value(userIDKey).(int)
            if ok {
              fmt.Println(userID)
            }
          }




---------- (Create child contexts) ----------

* context.WithCancel:

      ctx, cancel := context.WithCancel(<parent_context>)

          ctx   : Derived context (child context).
          cancel: Function you call to cancel the context.


      * Create a context that can be manually canceled.


      * When cancel() is called:

            * ctx.Done() is triggered.
            * ctx.Err() returns context.Canceled.
            * All go routines waiting on ctx.Done() are notified.
            * Cancellation propagates downward to child contexts.


      Ex:

          func main() {
            ctx, cancel := context.WithCancel(context.Background())

            go func() {
              <-ctx.Done()
              fmt.Println("Canceled:", ctx.Err())
            }()

            time.Sleep(5 * time.Second)
            cancel()

            time.Sleep(time.Second)
          }




* context.WithCancelCause:

      ctx, cancel := context.WithCancelCause(<parent_context>)


      To cancel with a reason.

          * Access the cause     : context.Cause(<context>)
          * To cancel the context: cancel(<custom_error>)


      Ex:

          func main() {
            ctx, cancel := context.WithCancelCause(context.Background())

            go func() {
              <-ctx.Done()

              fmt.Println("Canceled:", context.Cause(ctx))
            }()

            time.Sleep(5 * time.Second)
            cancel(errors.New("connetion lost"))

            time.Sleep(time.Second)
          }




* context.WithDeadline:

      * ctx, cancel := context.WithDeadline(<parent_context>, <time>)


      * Automatically cancels itself at a specific time or can be cancelled
        manually by calling cancel().


      * Propagates cancellation to all child goroutines using this context.


      Ex:

          func main() {
            ctx, cancel := context.WithDeadline(context.Background(), time.Now().Add(5*time.Second))

            go func() {
              time.Sleep(10 * time.Second)

              cancel()
            }()

            <-ctx.Done()
            fmt.Println(ctx.Err())
          }




* context.WithDeadlineCause:

      * ctx, cancel := context.WithDeadlineCause(<parent_context>, <time>, <custom_error>)


      * Used to cancel with cause.


      * You get the custom error only if the deadline is exceeded, but not with
        cancel().


      * If the deadline is already in the past, the context is canceled
        immediately.


      Ex:

          func main() {
            ctx, cancel := context.WithDeadlineCause(context.Background(), time.Now().Add(5*time.Second), errors.New("test error"))

            go func() {
              time.Sleep(10 * time.Second)

              cancel()
            }()

            <-ctx.Done()
            fmt.Println(context.Cause(ctx))
          }




* context.WithTimeout:

      * ctx, cancel := context.WithTimeout(<parent_context>, <duration>)


      * Automatically cancels itself after a fixed duration or can be cancelled
        manually by calling cancel().


      Ex:

          func main() {
            ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)

            go func() {
              time.Sleep(2 * time.Second)

              cancel()
            }()

            <-ctx.Done()
            fmt.Println(ctx.Err())
          }




* context.WithTimeoutCause:

      * ctx, cancel := context.WithTimeoutCause(<parent_context>, <duration>, <custom_error>)


      * To cancel with a cause.


      Ex:

          func main() {
            ctx, cancel := context.WithTimeoutCause(context.Background(), 5*time.Second, errors.New("test error"))

            go func() {
              time.Sleep(6 * time.Second)

              cancel()
            }()

            <-ctx.Done()
            fmt.Println(context.Cause(ctx))
          }




* context.WithValue:

      * ctx := context.WithValue(<parent_context>, <key>, <value>)


      * Used to attach request-scoped data to a context so that it can be passed
        across API boundaries and goroutines.


      Ex:

          type userIDKeyType string

          func main() {
            var userIDKey userIDKeyType = "userID"
            ctx := context.WithValue(context.Background(), userIDKey, 1)

            v, ok := ctx.Value(userIDKey).(int)
            if ok {
              fmt.Println(v)
            }
          }




* context.WithoutCancel:

      * Returns a context that ignores cancelation from parent.


      * Not using context.WithoutCancel:

            type userIDKeyType string

            func main() {
              parent, parentCancel := context.WithCancel(context.Background())

              var userIDKey userIDKeyType = "userID"
              child := context.WithValue(parent, userIDKey, 1)

              go func() {
                <-parent.Done()
                fmt.Println("Parent is cancelled")
              }()

              go func() {
                <-child.Done()
                fmt.Println("Child is cancelled")
              }()

              time.Sleep(2 * time.Second)
              parentCancel()

              time.Sleep(2 * time.Second)

              fmt.Println("End")
            }




      * Using context.WithoutCancel:

            func main() {
              parent, parentCancel := context.WithCancel(context.Background())

              child := context.WithoutCancel(parent)

              go func() {
                <-parent.Done()
                fmt.Println("Parent is cancelled")
              }()

              go func() {
                <-child.Done()
                fmt.Println("Child is cancelled")
              }()

              time.Sleep(2 * time.Second)
              parentCancel()

              time.Sleep(2 * time.Second)

              fmt.Println("End")
            }




* context.AfterFunc:

      * stop := context.AfterFunc(<context>, <callback>)


      * It schedules a function to run when the context's deadline expires, not when
        it is manually canceled.


      * Only make sense when used with:

            context.WithDeadline
            context.WithTimeout


      * The returned stop function can also prevent execution if called before the
        deadline.


      Ex:

          func test() {
            fmt.Println("Function is executed")
          }

          func main() {
            ctx, _ := context.WithTimeout(context.Background(), 5*time.Second)
            stop := context.AfterFunc(ctx, test)

            go func() {
              time.Sleep(6 * time.Second)

              stop()
            }()

            <-ctx.Done()
            fmt.Println("End")
          }

---------- Timers --------------------------------------------------------------

---------- (time.NewTimer) ----------

* Creation:

      timer := time.NewTimer(<duration>)




* timer.C:

      * It is a receive-only unbuffered channel:

            <-timer.C


      * It sends exactly one value (triggered time) after the specified duration
        and stays active (blocking) forever.




* Stop a timer:

      * stopped := timer.Stop():

            If stopped == true:

                <-timer.C is not accessed yet.


            If stopped == false:

                1. <-timer.C is already accessed. or,
                2. timer.Stop() is already executed.


      * <-timer.C is always active (blocking) forever even after timer.Stop().




* Reset a timer:

      * ok := timer.Reset(<new_duration>)

      * It always resets the timer.

            ok == true:

                <-timer.C is not accessed yet.


            ok == false:

                1. <-timer.C is already accessed, or
                2. timer.Stop() is already executed




Ex: Scheduling a delayed operation:

    func main() {
      var wg sync.WaitGroup

      timer := time.NewTimer(2 * time.Second)

      wg.Go(func() {
        <-timer.C

        fmt.Println("Delayed operation executed")
      })

      fmt.Println("Waiting...")

      wg.Wait()
    }




---------- (time.AfterFunc) ----------

* Returns a timer.


* Can use to execute a callback after the specified duration.


* Stop:

      stopped := timer.Stop()

          stopped == true:

              Function is not triggered yet.


          stopped == false:

              1. Function is already triggered. or,
              2. timer.Stop() is already executed


* Reset:

      * It always resets the timer.

      * ok := timer.Reset(<new_duration>)

            ok == true:

                Function is not triggered yet.


            ok == false:

                1. Function is already triggered, or
                2. timer.Stop() is already executed


Ex:

    func callback() {
      fmt.Println("Function is executed")
    }

    func main() {
      timer := time.AfterFunc(5*time.Second, callback)

      time.Sleep(2 * time.Second)

      stopped := timer.Stop()
      fmt.Println("Stopped:", stopped) // true

      value := timer.Reset(time.Second)
      fmt.Println(value) // false

      time.Sleep(2 * time.Second)
      fmt.Println("End")
    }




---------- (time.After) ----------

* Returns a read-only unbuffered channel.


* The channel receives current time after the duration.


* Cannot stop or reset it.


Ex:

    func main() {
      ch := make(chan int)

      go func() {
        time.Sleep(3 * time.Second)

        ch <- 1
      }()

      select {
      case v := <-ch:
        fmt.Println(v)

      case <-time.After(2 * time.Second):
        fmt.Println("Timeout")
      }
    }

---------- Tickers -------------------------------------------------------------

* Used to execute something repeatedly at a fixed time interval.


* It sends the current time on the channel "C" every N duration until you stop
  it.


* Does not stop automarically. You must stop it manually.

      * ticker.Stop()

            * No more values are received from ticker.C

            * Channel is active (blocking) forever

            * No return values.


* Can reset:

      * ticker.Reset(<new_duration>)

      * It always resets the ticker.

      * No return values.


Ex:

    func main() {
      ticker := time.NewTicker(1 * time.Second)

      for t := range ticker.C {
        fmt.Println(t)
      }
    }


* Combining tickers with timers:

      func main() {
        ticker := time.NewTicker(time.Second)
        defer ticker.Stop()

        stop := time.After(5 * time.Second)

        for {
          select {
          case tick := <-ticker.C:
            fmt.Println(tick)

          case <-stop:
            fmt.Println("Stopping ticker...")
            return
          }
        }
      }

---------- Worker Pool ---------------------------------------------------------

* A worker pool is a design pattern used to manage a group of workers (goroutines).


* In a worker pool pattern:

      * Workers control concurrency.
      * Buffer size controls pressure.


* Most common default:

      Buffer size == numWorkers


Ex 1:

    type Job struct {
      ID int

      // Place fields that are needed to carry job data
    }

    func worker(id int, jobs <-chan Job, results chan<- string) {
      for job := range jobs {
        // Simulated work
        time.Sleep(time.Second)
        data := fmt.Sprintf("Worker %d processed job %d", id, job.ID)

        // Send calculated results to the channel
        results <- data
      }
    }

    func main() {
      const numWorkers = 3
      const numJobs = 10

      jobs := make(chan Job, numJobs)
      results := make(chan string, numJobs)

      // Create workers
      for id := range numWorkers {
        go worker(id, jobs, results)
      }

      // Send values to the jobs channel (In real world we do not limit jobs)
      for id := range numJobs {
        job := Job{
          ID: id,
        }

        jobs <- job
      }
      close(jobs)

      // Access results
      for range numJobs {
        result := <-results

        fmt.Println(result)
      }
    }




Ex 2:

    type TicketRequest struct {
      UserID     int
      NumTickets int
      Cost       int
    }

    func ticketProcessor(requests <-chan TicketRequest, results chan<- int) {
      for req := range requests {
        fmt.Printf("Processing %d ticket(s) of user %d with total cost %d\n", req.NumTickets, req.UserID, req.Cost)

        // Simulate processing time
        time.Sleep(time.Second)

        results <- req.UserID

      }
    }

    func main() {
      const numTicketProcessors = 3 // workers
      const numRequests = 5

      requests := make(chan TicketRequest, numRequests)
      results := make(chan int, numRequests)

      // Start workers
      for range numTicketProcessors {
        go ticketProcessor(requests, results)
      }

      // Send ticket requests
      const ticketPrice = 4
      for i := range numRequests {
        userID := i + 1
        numTickets := (i + 1) * 2
        cost := numTickets * ticketPrice

        requests <- TicketRequest{UserID: userID, NumTickets: numTickets, Cost: cost}
      }
      close(requests)

      for range numRequests {
        fmt.Printf("Ticket for userID %d is processed successfully!\n", <-results)
      }
    }

---------- Wait Groups ---------------------------------------------------------

* A wait group is a synchronization primitive that lets one goroutine "wait" for
  a group of other goroutines to finish.


* Think of it as:

      Run these goroutines in parallel, and don't continue until all of them are
      done.


* Creation:

      var wg sync.WaitGroup


* Methods:

      wg.Add(n)    : Increase counter by n.

      wg.Done()    : Decrease counter by 1. ( wg.Done() == wg.Add(-1) )

      wg.Wait()    : Block until counter = 0.

      wg.Go(<func>): It calls Add(1), launches the goroutine, and ensures Done()
                     runs when the function returns.




* Without wait group:

      func worker() {
        fmt.Println("Worker function is executed")
      }

      func main() {
        go worker()
      }

      * main() exits -> program ends -> goroutine may never run.




* Solution:

      func worker(id int, wg *sync.WaitGroup) {
        defer wg.Done()

        fmt.Printf("Worker %d is executed\n", id)
      }

      func main() {
        const numWorkers = 3

        var wg sync.WaitGroup

        wg.Add(numWorkers)
        for id := range numWorkers {
          go worker(id, &wg)
        }

        wg.Wait()
      }


      or, with wg.Go


      func worker(id int) {
        fmt.Printf("Worker %d is executed\n", id)
      }

      func main() {
        const numWorkers = 3

        var wg sync.WaitGroup

        for id := range numWorkers {
          wg.Go(func() {
            worker(id)
          })
        }

        wg.Wait()
      }




* With channels:

      func worker(id int, results chan<- int) {
        fmt.Printf("Worker %d is starting\n", id)

        time.Sleep(time.Second)

        results <- id * 2

        fmt.Printf("Worker %d is finished\n", id)
      }

      func main() {
        const numWorkers = 3

        var wg sync.WaitGroup

        results := make(chan int, numWorkers)

        for id := range numWorkers {
          wg.Go(func() {
            worker(id, results)
          })
        }

        go func(wg *sync.WaitGroup) {
          wg.Wait()

          close(results)
        }(&wg)

        for result := range results {
          fmt.Println("Result:", result)
        }
      }

---------- Mutual Exclusion (Mutexes) ------------------------------------------

* A mutex is a synchronization primitive used to protect shared data from being
  accessed by multiple goroutines at the same time.


* If two goroutines read/write the same variable at the same time, you get a
  data race.


* Often used in structs.


Ex:

    type Counter struct {
      Mu    sync.Mutex
      Value int
    }

    func (c *Counter) increment() {
      c.Mu.Lock()
      defer c.Mu.Unlock()

      c.Value++
    }

    func main() {
      var wg sync.WaitGroup
      counter := &Counter{}

      numGoroutines := 10

      for range numGoroutines {
        wg.Go(func() {
          for range 1000 {
            counter.increment() // (correct)

            // counter.Value++ (Wrong)
          }
        })
      }

      wg.Wait()

      fmt.Println("Count:", counter.Value)
    }




* mu.TryLock():

      * mu.TryLock() == true means:

            * No other goroutine was holding the mutex.

            * You successfully acquired the lock.

            * Only your goroutine can access the protected data.


      * Once you hold the lock, others will block or fail mu.TryLock() until you
        call mu.Unlock().


      * Practical usage:

            if mu.TryLock() {
              defer mu.Unlock()

              // safe access
            }

---------- Atomic Counters -----------------------------------------------------

* Atomic counters are variables whose read-modify-write operations happen
  atomically, meaning no other goroutine can observe or interfere with the
  operation halfway through.


* Operations:

      Add : Add a given value


      CompareAndSwap :

          Compare the stored value with the given value. If both are equal, then
          store the new value and returns the old value.


      Load : Get the stored value


      Store : To store a new value


      Swap : To store a new value and returns the old value




Ex:

    func main() {
      var counter atomic.Int64

      counter.Store(10)
      counter.CompareAndSwap(10, 20) // 20

      fmt.Println(counter.Load())
    }

---------- Rate Limiting -------------------------------------------------------

---------- (Token Bucket Algorithm) ----------

* How it works:

      * A bucket (channel) holds tokens.
      * Tokens are added at a fixed rate (tokens per second).
      * Each request consumes tokens.
      * If enough tokens exist --> request is allowed.
      * If no tokens --> request is rejected.


* Basic implementation:

      type TokenBucket struct {
        tokens     chan struct{}
        refillRate time.Duration // duration to add 1 token
      }

      func (tb *TokenBucket) refill() {
        ticker := time.NewTicker(tb.refillRate)
        defer ticker.Stop()

        for range ticker.C {
          select {
          case tb.tokens <- struct{}{}:

          default:
          }
        }
      }

      func (tb *TokenBucket) Allow() bool {
        select {
        case <-tb.tokens:
          return true

        default:
          return false
        }
      }

      func NewTokenBucket(capacity int, refillRate time.Duration) *TokenBucket {
        tb := &TokenBucket{
          tokens:     make(chan struct{}, capacity),
          refillRate: refillRate,
        }

        for range capacity {
          tb.tokens <- struct{}{}
        }

        go tb.refill()

        return tb
      }

      func main() {
        rateLimiter := NewTokenBucket(5, 200*time.Millisecond)

        for range 100 {
          if rateLimiter.Allow() {
            fmt.Println("Allowed")
          } else {
            fmt.Println("Denied")
          }

          time.Sleep(30 * time.Millisecond)
        }
      }




* Weaknesses:

      * Complexity in distributed systems.
      * Not ideal for precise rate limiting.




---------- (Fixed Window Counter) ----------

* How it works:

      * Start a time window
      * Count incoming requests
      * Check against the limit
      * Reset the counter


* Basic implementation:

      type FixedWindowCounter struct {
        mu        sync.Mutex
        count     int
        limit     int
        window    time.Duration
        resetAt   time.Time
      }

      func (fwc *FixedWindowCounter) Allow() bool {
        fwc.mu.Lock()
        defer fwc.mu.Unlock()

        now := time.Now()
        if now.After(fwc.resetAt) {
          fwc.resetAt = now.Add(fwc.window)
          fwc.count = 0
        }

        if fwc.count < fwc.limit { // Not <=
          fwc.count++
          return true
        }

        return false
      }

      func NewFixedWindowCounter(limit int, window time.Duration) *FixedWindowCounter {
        return &FixedWindowCounter{
          limit:  limit,
          window: window,
        }
      }

      func main() {
        fwc := NewFixedWindowCounter(5, time.Second)

        for range 100 {
          if fwc.Allow() {
            fmt.Println("Allowed")
          } else {
            fmt.Println("Denied")
          }

          time.Sleep(30 * time.Millisecond)
        }
      }


* Weaknesses:

      * Burst problem at window edges:

            A client can send 5 requests at the end of one window and 5 more at
            the start of the next, effectively making 10 requests in a short
            time.

      * Not smooth:

            Rate limiting is "chunky" per window.




---------- (Leaky Bucket Algorithm) ----------

* How it works:

      * Incoming requests = water poured into the bucket:

            * Each request is one unit of water.
            * When a request arrives, it is added to the bucket.


      * Constant leak rate:

            * This never changes, no matter how many requests arrive.


      * Bucket capacity limit:

            * The bucket can only hold up to N units.

            * If a new request arrives when the bucket is full, the request is
              rejected (dropped).


* Basic implementation:

      type LeakyBucket struct {
        mu        sync.Mutex
        capacity  int     // Max water level
        level     float64 // Current water level
        leakRate  float64 // Units per second
        lastCheck time.Time
      }

      func (lb *LeakyBucket) Allow() bool {
        lb.mu.Lock()
        defer lb.mu.Unlock()

        now := time.Now()
        elapsedDuration := now.Sub(lb.lastCheck).Seconds()
        lb.lastCheck = now

        // Leak water
        lb.level -= elapsedDuration * lb.leakRate
        if lb.level < 0 {
          lb.level = 0
        }

        // Try to add one request
        if lb.level+1 > float64(lb.capacity) {
          return false
        }

        lb.level++
        return true
      }

      func NewLeakyBucket(capacity int, leakRate float64) *LeakyBucket {
        return &LeakyBucket{
          capacity:  capacity,
          leakRate:  leakRate,
          lastCheck: time.Now(),
          // level: 0 (Initially the water level is 0)
        }
      }

      func main() {
        rateLimiter := NewLeakyBucket(5, 5)

        for range 100 {
          if rateLimiter.Allow() {
            fmt.Println("Allowed")
          } else {
            fmt.Println("Denied")
          }

          time.Sleep(30 * time.Millisecond)
        }
      }


* Weaknesses:

      * No burst tolerance.
      * Unnecessary request drops.
      * Fixed output rate only.
      * Not fair for mixed traffic.
      * Hign latency under load.

---------- Sorting -------------------------------------------------------------

* Sort an integer slice:

      numbers := []int{5, 4, 3, 1, 2, 4}
      sort.Ints(numbers)


* Sort a string slice:

      stringSlice := []string{"john", "anthony", "steve", "victor", "walter"}
      sort.Strings(stringSlice)




---------- (sort.Interface) ----------

type Interface interface {
  Len() int
  Less(i, j int) bool
  Swap (i, j int)
}


1. Len() int:

    Returns the number of elements.


2. Less(i, j int) bool:

    Defines ordering logic.

        * Return true if element at index i should come before element at
          index j.

        * This is where you decide ascending, descending, custom rules, etc.


3. Swap(i, j int):

    Swaps elements at positions i and j.

        * Sorting algorithm rearranges elements using this.




Ex (Method 1):

    type Person struct {
      Name string
      Age  int
    }

    type ByAge []Person

    func (a ByAge) Len() int {
      return len(a)
    }

    func (a ByAge) Less(i, j int) bool {
      return a[i].Age < a[j].Age
    }

    func (a ByAge) Swap(i, j int) {
      a[i], a[j] = a[j], a[i]
    }

    func main() {
      people := []Person{
        {Name: "Alice", Age: 30},
        {Name: "Bob", Age: 25},
        {Name: "Anna", Age: 35},
      }

      sort.Sort(ByAge(people))
      fmt.Println(people)
    }




Ex (Method 2: Recommended):

    type Person struct {
      Name string
      Age  int
    }

    type By func(p1, p2 *Person) bool

    func (by *By) Sort(people []Person) {
      ps := PersonSorter{
        people: people,
        by:     *by,
      }

      sort.Sort(&ps)
    }

    type PersonSorter struct {
      people []Person
      by     By
    }

    func (ps *PersonSorter) Len() int {
      return len(ps.people)
    }

    func (ps *PersonSorter) Less(i, j int) bool {
      return ps.by(&ps.people[i], &ps.people[j])
    }

    func (ps *PersonSorter) Swap(i, j int) {
      ps.people[i], ps.people[j] = ps.people[j], ps.people[i]
    }

    func main() {
      people := []Person{
        {Name: "Alice", Age: 30},
        {Name: "Bob", Age: 25},
        {Name: "Anna", Age: 35},
      }

      age := func(p1, p2 *Person) bool {
        return p1.Age < p2.Age
      }

      byAge := By(age)
      (&byAge).Sort(people)

      fmt.Println(people)
    }




---------- (sort.Slice) ----------

* It is a short form of above operation.


Ex:

    func main() {
      people := []string{"Anna", "John", "Bob", "Alice"}

      sort.Slice(people, func(i, j int) bool {
        return people[i] < people[j]
      })

      fmt.Println(people)
    }

---------- Testing  ------------------------------------------------------------

* To create tests for a go file, create a new go file in same package called:

      <original_file_name>_test.go


* There are 2 ways to specify package in that go test file:

      1. package <pkg>:

          * Can access all items in current package including private items.


      2. package <pkg>_test:

          * Consider as a seperate package.
          * To use items in current package, we must import it.
          * Only public items are allowed.


* All test functions must:

      * Start with Test.
      * Contain "t *testing.T" parameter.


      * Otherwise those functions are ignored.


      Ex:

          package sample

          import (
            "testing"
          )

          func TestAdd(t *testing.T) {
            expected := 30
            result := Add(10, 20) // Function to test from same package

            if result != expected {
              t.Error("Expected:", expected, ", got:", result)
            }
          }


* Test against multiple inputs (Table Driven Test):

      Ex:

          package sample

          import (
            "testing"
          )

          func TestAddTableDriven(t *testing.T) {
            test := []struct{ a, b, expected int }{{1, 2, 3}, {10, 20, 30}, {100, 200, 300}}

            for _, test := range test {
              result := Add(test.a, test.b)  // Function to test from same package
              if result != test.expected {
                t.Error("Expected:", test.expected, ", got:", result)
              }
            }
          }


* To get better test failed outputs use subtests:

      Ex:

          package sample

          import (
            "fmt"
            "testing"
          )

          func TestAddTableDriven(t *testing.T) {
            test := []struct{ a, b, expected int }{{1, 2, 3}, {10, 20, 30}, {100, 200, 400}}

            for _, test := range test {
              t.Run(fmt.Sprintf("Add(%d,%d)", test.a, test.b), func(t *testing.T) {
                result := Add(test.a, test.b)

                if result != test.expected {
                  t.Errorf("Expected: %d, Got: %d", test.expected, result)
                }
              })
            }
          }


* To run all tests in a module:

      go test ./...


* To run tests in a specific package(folder):

      go test ./<pkg>

        if err != nil {
          fmt.Println("Error waiting")

        }

        fmt.Println("end")
      }

---------- Signals -------------------------------------------------------------

* Signals are buffered channels with type os.Signal.

      sigs := make(chan os.Signal, 1)


* To send an os signal:

      kill -s <signal_name> <pid>

      Ex: kill -s SIGINT <pid>


Ex:

    func main() {
      pid := os.Getpid() // To get the process id
      fmt.Println("pid:", pid)

      sigs := make(chan os.Signal, 1)

      // Listening and notify the channel on interrupt or terminate signals
      signal.Notify(sigs, syscall.SIGINT, syscall.SIGTERM)

      go func() {
        sig := <-sigs
        fmt.Println("Received signal:", sig)
        fmt.Println("Graceful exit")
        os.Exit(0)
      }()

      for {
        fmt.Println("Working...")
        time.Sleep(time.Second)
      }
    }


    * This program is exit gracefully if we kill it with SIGTERM or SIGINT
      signals.

    * If we send an another signal, program is terminated. But it is not a
      graceful termination.


* Handle each signal seperately:

      func main() {
        pid := os.Getpid()
        fmt.Println("pid:", pid)

        sigs := make(chan os.Signal, 1)

        // Listening and notify the channel on interrupt or terminate signals
        signal.Notify(sigs, syscall.SIGINT, syscall.SIGTERM)

        go func() {
          sig := <-sigs

          switch sig {
          case syscall.SIGINT:
            fmt.Println("SIGINT")
          case syscall.SIGTERM:
            fmt.Println("SIGTERM")
          }

          fmt.Println("Graceful exit")
          os.Exit(0)
        }()

        for {
          fmt.Println("Working...")
          time.Sleep(time.Second)
        }
      }

---------- Reflection ----------------------------------------------------------

* Reflection allows a program to inspect and manipulate variables, types, and
  values at runtime, even when their type is not known at compile time.


* The reflection features are provided by the package: reflect


* Everything revolves around two core types:

      1. reflect.Type:

            * Return type of: reflect.TypeOf().

            * Static description of the value.

            * No data, just structure.

            * Immutable metadata.

            * Ask: What kind of thing is that?

            * Usage:

                  t := reflect.TypeOf(value) or,

                  t := reflect.ValueOf(value).Type() or,

                  t := reflect.ValueOf(&value).Elem().Type()


      2. reflect.Value:

            * Return type of: reflect.ValueOf().

            * Actual runtime data.

            * Can read or modify (if allowed).

            * Carries the value itself.

            * Ask: What is the actual value and can I use/change it?

            * Usage:

                  v := reflect.ValueOf(value)         : If you don't want to modify the value.

                  v := reflect.ValueOf(&value).Elem() : If you want to modify the value. Elem() dereference the provided pointer.




* To modify a value, we have to provide it as a pointer.

      func main() {
        x := 10
        v := reflect.ValueOf(&x).Elem()

        v.SetInt(20)

        fmt.Println(v) // 20
      }




* Some basic methods:

      func main() {
        type MyInt int
        var value MyInt = 10

        v := reflect.ValueOf(value)

        fmt.Println(v)          // 10
        fmt.Println(v.Type())   // main.MyInt
        fmt.Println(v.Kind())   // int
        fmt.Println(v.IsZero()) // false
      }


      * Type: Exact type

      * Kind: Underlying category (int, struct, etc)

      * IsZero: Is this value the default zeroth value for its type?




* Check the type of a value is int or not:

      func main() {
        type MyInt int

        var a MyInt = 20
        var b any = a

        v := reflect.ValueOf(b)

        isInt := v.Kind() == reflect.Int
        fmt.Println(isInt)
      }




---------- (Reflects With Structs) ----------

type Person struct {
	Name string `json:"name,omitempty"`
	Age  int    `json:"age,omitempty"`
}

func main() {
	p := Person{
		Name: "John Cena",
		Age:  49,
	}

	v1 := reflect.ValueOf(&p).Elem()
	v2 := reflect.ValueOf(p)

	t := reflect.TypeOf(p)

	// Number of fields
	fmt.Println(v1.NumField()) // 2

	// * Access values of each fields
	// * Order is same as declaration order of the struct fields
	fmt.Println(v1.Field(0))            // John Cena
	fmt.Println(v1.Field(1))            // 49
	fmt.Println(v1.FieldByName("Name")) // John Cena
	fmt.Println(v1.FieldByName("Age"))  // 49

	// Check the type and kind of a field
	fmt.Println(v1.Field(0).Type())                // int
	fmt.Println(v1.Field(0).Kind() == reflect.Int) // true

	// Check a value can be changed or not
	fmt.Println(v1.CanSet())          // true
	fmt.Println(v2.CanSet())          // false
	fmt.Println(v1.Field(1).CanSet()) // true
	fmt.Println(v2.Field(1).CanSet()) // false

	// Access json tags
	fmt.Println(t.Field(0).Tag.Get("json")) // age,omitempty

	// Change field values
	v1.FieldByName("Name").Set(reflect.ValueOf("Anna Silva"))
	v1.FieldByName("Age").SetInt(50)
	fmt.Println(v1) // {Anna Silva 50}
}

---------- RWMutex -------------------------------------------------------------

* Read Write Mutex (sync.RWMutex)


* Read:

      Read Lock   : RLock()
      Read Unlock : RUnlock()

      * Multiple readers are allowed simultaneously.
      * No writers are allowed while reading.


* Write:

      Write Lock   : Lock()
      Write Unlock : Unlock()

      * Only one writer is allowed.
      * No readers are allowed while writing.


Ex:
    var (
      rwmu  sync.RWMutex
      count int = 0
    )

    func readCounter() {
      rwmu.RLock()
      defer rwmu.RUnlock()

      fmt.Println("Read Counter:", count)
    }

    func writeCounter(value int) {
      rwmu.Lock()
      defer rwmu.Unlock()

      count = value
    }

---------- sync.NewCond --------------------------------------------------------

* sync.NewCond creates a condition variable. A condition variable lets goroutines
  wait until some condition becomes true, while coordinating with a mutex.


* Type of a condition variable    : *sync.Cond (Always use as a pointer)

* Create a new condition variable : sync.NewCond(<sync.Locker>)

* sync.Locker is an interface:

      type Locker interface {
        Lock()
        Unlock()
      }

* We can provide both sync.Mutex and sync.RWMutex for sync.NewCond, but for
  RWMutex, only writer lock ( Lock() ) and writer unlock ( Unlock() ) are allowed.


* cond.Wait():

      * Blocking.

      * Always use inside a Lock() with a for loop.

      * When it is called:

            1. Unlock mutex
            2. Waiting for the signal/broadcase

      * When a signal happens:

            1. Lock the mutex again
            2. execute the code further.


* cond.Signal():

      * Non-blocking.

      * Wakes up only one waiting goroutine.

      * Always recommended to use inside a lock.


* cond.Broadcast():

      * Non-blocking.

      * Wakes up all waiting goroutine.

      * But the lock is acquired only one goroutine at a time.

      * Always recommended to use inside a lock.


Ex:

    const bufferSize = 5

    type Buffer struct {
      Items []int
      mu    sync.Mutex
      cond  *sync.Cond
    }

    func (b *Buffer) produce(item int) {
      b.mu.Lock()
      defer b.mu.Unlock()

      for len(b.Items) == bufferSize {
        b.cond.Wait() // wait until consumer consume
      }

      b.Items = append(b.Items, item)
      fmt.Println("Produced:", item)
      b.cond.Signal() // Signal to a waiting goroutine in the consumer
    }

    func (b *Buffer) consume() int {
      b.mu.Lock()
      defer b.mu.Unlock()

      for len(b.Items) == 0 {
        b.cond.Wait() // waiting until producer produce
      }

      item := b.Items[0]
      b.Items = b.Items[1:]
      fmt.Println("Consumed:", item)
      b.cond.Signal() // Signal to a waiting goroutine in the producer

      return item
    }

    func NewBuffer(size int) *Buffer {
      b := &Buffer{
        Items: make([]int, 0, size),
      }
      b.cond = sync.NewCond(&b.mu)
      return b
    }

    func producer(b *Buffer) {
      for i := range 10 {
        b.produce(i + 100)
        time.Sleep(100 * time.Millisecond)
      }
    }

    func consumer(b *Buffer) {
      for range 10 {
        b.consume()
        time.Sleep(200 * time.Millisecond)
      }
    }

    func main() {
      buffer := NewBuffer(bufferSize)
      var wg sync.WaitGroup

      wg.Go(func() {
        producer(buffer)
      })

      wg.Go(func() {
        consumer(buffer)
      })

      wg.Wait()
    }

---------- sync.Pool -----------------------------------------------------------

* Its main purpose is to reuse objects to reduce GC overhead and improve
  performance in high-throughput applications.


* Think of it as a "bucket" where you can get an object if available, or put it
  back for reuse later.


* We use Get() method to get an object from the pool in LIFO order. (like a stack)


* We use Put() method to push objects to the pool.


Ex:

    type Person struct {
      Name string
      Age  int
    }

    func main() {
      pool := sync.Pool{
        New: func() any { // Calls when you do Get() and the pool has no objects to return, otherwise panic
          fmt.Println("Creating a new Person")
          return &Person{}
        },
      }

      // Get an object from the pool
      person1 := pool.Get().(*Person)
      person1.Name = "John"
      person1.Age = 30

      fmt.Println(person1)

      pool.Put(person1)

      person2 := pool.Get()
      fmt.Println(person2)
    }

---------- How Internet Works --------------------------------------------------

* Internet:

      A global network of inter-connected computers that communicate using
      standardized protocols.




* A web request's journey:

      1. Entering a URL.


      2. DNS Lookup, DNS Server Interaction.


      3. Establishing a TCP connection.

            * Browser sends a TCP SYN (synchronize) packet to the server.

            * Server responds with a SYN-ACK (synchronize-acknowledge) packet.

            * Browser sends an ACK (acknowledge) packet, completing the
              three-way handshake.


      4. Sending an HTTP request.


      5. Server Processing and Response.


      6. Rendering the web page.




* Network layers involved:

      1. Application layer:

            * Protocols       : HTTP, HTTPS, DNS

            * Responsible for : High-level APIs, resource sharing, and remote
              file access.


      2. Transport layer:

            * Protocols       : TCP, UDP

            * Responsible for : End-to-end communication, error checking, and
              data flow control.


      3. Internet Layer:

            * Protocols       : IP
            * Responsible for : Addressing, routing, and packet forwarding.


      4. Link layer:

            * Protocols       : Ethernet, Wi-Fi
            * Responsible for : Physical addressing and media access control.




* URL vs URI:

      * URI : Uniform Resource Identifier.
      * URL : Uniform Resource Locator.
      * URN : Uniform Resource Name.


      * Components of a URL:

            * Schema
            * Host
            * Port
            * Path
            * Query
            * Fragment

---------- Cookies -------------------------------------------------------------

* A cookie is a small piece of data that a server sends to a client, which the
  client stores and automatically sends back with every subsequent request to
  that server.


* A cookie is also a header.


* Send cookies to client from the server:

      http.SetCookie(w, &http.Cookie{...})

      * Cookies are sent to client with header: "set-cookie".
      * Single response can contains multiple "set-cookie" headers.


* Access cookies come from clients in the server:

      cookie, err := r.Cookie(<cookie_name>)
      if err != nil {
        fmt.Println("cookie not found")
      } else {
        fmt.Println(cookie.Value)
      }

      * All cookies are come from the client with single header: "Cookie".


* Properties:

      1. Name: Name of the cookie


      2. Value: Value of the cookie


      3. Path:

            * Controls which URL paths can access the cookie.

            * A cookie is sent only if the request path starts with the
              cookie's path.

            * Default: Request path.

            * If the cookie's path matches, the browser sends it with all HTTP
              methods (GET, POST, PATCH, DELETE, etc).


      4. Domain:

            * Controls which domains/subdomains receive the cookie.

            * Most of the time we don't set manually. It will be provided
              automarically.

            * you can only set cookie for:

                  * The same domain, or
                  * A parent domain

                  Ex: If the request comes from "api.example.com", You can set
                      Domain=api.example.com or Domain=example.com

            * Default: Domain of the current request.


      5. Expires & MaxAge:

            * Expires: Exact timestamp of expiration.

            * MaxAge : Relative time in seconds.

            * If both are provided, MaxAge is considered.

            * If both are not provided:

                  * Expires = Session

                  * The cookie is deleted when the browser session ends
                    (browser close), not when the tab closes.


      6. HttpOnly:

            * If true, prevents accessing the cookie using javascript.

            * Default: false


      7. Secure:

            * If true, sends the cookie only over HTTPS.

            * Default: false


      8. SameSite:

            * SameSite controls whether cookies are sent when a request
              originates from another site (different origin).

            * A site is basically: scheme + domain

                  Ex:
                      Consider: https://example.com

                      https://example.com     -> SameSite

                      https://api.example.com -> SameSite (Subdomain counts as same site)

                      https://evil.com        -> Different site


            * Values:

                  http.SameSiteStrictMode:

                      Cookie is sent only when request originates from the same
                      site.

                  http.SameSiteLaxMode:

                      Cookie is sent for same site requests and top-level
                      cross-site GET navigation, but blocks them for background
                      or unsafe cross-site requests to help prevent cross-site
                      request forgery.

                  http.SameSiteNoneMode:

                      Cookie is sent in all requests, including cross-site.


            * Default: http.SameSiteDefaultMode

                  Browser sets default: Lax


* For general use:

      * Name
      * Value
      * Path
      * MaxAge
      * Secure
      * HttpOnly

---------- Create A HTTP Server In GO ------------------------------------------

      func main() {

        err := http.ListenAndServe(fmt.Sprintf(":%d", port), nil)
        if err != nil {

---------- Server-Sent Events (SSE) --------------------------------------------

* SSE is a server-to-client stream of events over HTTP.


* Server (Go):

      func main() {
        http.HandleFunc("/stream", func(w http.ResponseWriter, r *http.Request) {
          // Set Headers
          w.Header().Set("Content-Type", "text/event-stream") // required
          w.Header().Set("Cache-Control", "no-cache")         // not required, but recommended
          w.Header().Set("Connection", "keep-alive")          // not required, but recommended
          w.Header().Set("Access-Control-Allow-Origin", "*")  // Needed if client is cross-origin

          // Ensure we can flush
          flusher, ok := w.(http.Flusher)
          if !ok {
            http.Error(w, "Streaming unsupported", http.StatusInternalServerError)
            return
          }

          ticker := time.NewTicker(2 * time.Second)
          defer ticker.Stop()

          id := 1
          for {
            select {
            case <-r.Context().Done():
              fmt.Println("Client disconnected")
              return

            case <-ticker.C:
              _, err := fmt.Fprintf(w, "Hello %d from server", id)
              if err != nil {
                fmt.Println("Write error:", err)
                return
              }
              id++

              flusher.Flush()
            }
          }
        })

        const port = 3000
        fmt.Println("Server is running on port:", port)
        err := http.ListenAndServe(fmt.Sprintf(":%d", port), nil)
        if err != nil {
          log.Fatalln("Error starting server:", err)
        }
      }




* Client (Curl):

      curl -N http://localhost:3000/stream

      -N : No buffer




* Client (Dart):

      import 'dart:convert';

      import 'package:http/http.dart' as http;

      void main() async {
        final client = http.Client();
        final req = http.Request("GET", Uri.parse("http://localhost:3000/stream"));

        final resp = await client.send(req);
        resp.stream.transform(utf8.decoder).listen((event) {
          print(event);
        });
      }

---------- SSL/TLS Certificates ------------------------------------------------

* SSL certificates are called SSL/TLS because TLS replaced SSL, but the old name
  "SSL certificate" remained in common use.

      SSL: Secure Sockets Layer
      TLS: Transport Layer Security




* How TLS encryption works:

      1. Client connect to server.
      2. Server sends its certificate with the public key.
      3. Client generates a random session key (symmetric key).
      4. Client encrypts the session key with server's public key and send it.
      5. Server decrypts with its private key. Both now have the same session key.
      6. All further communication is encrypted with the session key (fast symmetric encryption).




* Generate the TLS certificate and the private key for local development:

      * Install the tool:

            mkcert (sudo pacman -S mkcert --needed)


      * Run following command onece to install the local CA:

            mkcert -install

            CA: Certificate Authority


      * To generate certificates for your domain:

            mkcert <file_name> (Ex: mkcert localhost)

            It creates 2 files:

                1. localhost-key.pem (Private key)
                2. localhost.pem (Certificate)


      * For Postman:

            * Goto settings --> App settings --> Certificates

            * Toggle on CA certificates button and enter the following command:

                  mkcert -CAROOT

            * That command shows a location. In that location there is a file
              called "rootCA.pem". Add that file to the CA certificates PEM file

            * Then add Client certificates.

---------- http Package --------------------------------------------------------

---------- (http.Handler) ----------

* Definition:

      type Handler interface {
        ServeHTTP(http.ResponseWriter, *http.Request)
      }


* A handler is something that can handle an HTTP request.


* Anything that implements "ServeHTTP" method is a handler.


* When an HTTP request comes in, Go's server calls:

      handler.ServeHTTP(w, r)

          w -> http.ResponseWriter (Used to send response)
          r -> http.Request (Incoming request data)

      Ex:

          type MyHandler struct{}

          func (h *MyHandler) ServeHTTP(w http.ResponseWriter, r *http.Request) {
            fmt.Println("ServeHTTP of MyHandler is called")
          }

          func main() {
            handler := &MyHandler{}

            server := &http.Server{
              Addr:    ":3000",
              Handler: handler,
            }

            server.ListenAndServe()
          }




---------- (http.HandlerFunc) ----------

* Definition:

      type HandlerFunc func(ResponseWriter, *Request)

      func (f HandlerFunc) ServeHTTP(w ResponseWriter, r *Request) {
        f(w, r)
      }


* Any HandlerFunc type function is a handler:

      func handleRequest(w http.ResponseWriter, r *http.Request) {
        fmt.Println("request is handled")
      }

      func main() {
        var handler http.HandlerFunc = handleRequest

        server := &http.Server{
          Addr:    ":3000",
          Handler: handler,
        }

        server.ListenAndServe()
      }


* We use HandlerFunc type to convert any function with
  func(http.ResponseWriter, *http.Request) signature to a handler.




---------- (http.ServeMux) ----------

* It is a custom handler struct provided by http package.


* To create an instance of that struct:

      mux := http.NewServeMux()


* How ServeMux matches routes:

      1. Exact:

            /users

            matches only /users


      2. Prefix:

            /users/

            /users     ---> matches if not defined
            /users/    ---> matches if not defined
            /users/123 ---> matches if not defined


* mux.Handle:

      * Instead of responding every request, we can use mux.Handle to respond
        only to a specific url patterns.

            mux.Handle(<pattern>, <handler>)


      * When a request arrives, the server always calls:

            mux.ServeHTTP(w, r)


      * Inside mux.ServeHTTP:

            * It matches the request URL path against registered patterns.

            * Finds the corresponding <handler> you registered with mux.Handle.

            * Then it delegates:

                  handler.ServeHTTP(w, r)


* mux.HandleFunc:

      * Same idea of mux.Handle, but use a function with
        func(http.ResponseWriter, *http.Request) signature.

            mux.HandleFunc(<pattern>, <func>)


      * Internally same as:

            mux.Handle(<pattern>, http.HandlerFunc(<func>))

---------- CORS ----------------------------------------------------------------

* CORS: Cross-Origin Resource Sharing.


* SOP: Same Origin Policy

      * Deny all cross-origin access by default.
      * Browsers are SOP.


* CORS    : Controls who can read responses.
* SameSite: Controls when cookies are sent.


* CORS is a browser security mechanism that controls how web pages can request
  resources from a different origin than the one that served the page.


* Process:

      * Suppose you have already logged in to your bank and the authorization
        token is saved as cookie (https://bank.com).

      * Then you visit a malicious website called https://evil.com.

      * This malicious site executes following code in your browser:

            fetch("https://bank.com/users", {
              method: "PUT",
              headers: {
                "Authorization": "Bearer ..."
              }
            })

      * Now the browser sends preflight request with following informations:

            OPTIONS /users HTTP/1.1
            Origin: https://evil.com
            Access-Control-Request-Method: PUT
            Access-Control-Request-Headers: Authorization


            * Actual PUT request is not sent yet.
            * No sensitive action yets.
            * Browser is checking permission first.

      * If following informations are included in server response, the browser
        allows to make the actual request. If not browser blocks the actual
        request.

            * Access-Control-Allow-Origin: https://evil.com
            * Access-Control-Allow-Methods: PUT
            * Access-Control-Allow-Headers: Authorization


* Sample CORS middleware:

      var allowedOrigins = map[string]bool{
        "https://app.example.com": true,
        "https://app.example.org": true,
      }

      func Cors(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
          origin := r.Header.Get("Origin")

          if allowedOrigins[origin] {
            w.Header().Set("Access-Control-Allow-Origin", origin)
            w.Header().Set("Vary", "Origin") // important
            w.Header().Set("Access-Control-Allow-Credentials", "true")
          }

          // handle preflight
          if r.Method == http.MethodOptions {
            w.Header().Set("Access-Control-Allow-Methods", "GET, POST, PATCH, DELETE")
            w.Header().Set("Access-Control-Allow-Headers", "Content-Type, Authorization")
            return
          }

          next.ServeHTTP(w, r)
        })
      }


* Preflight request:

      * Before sending certain cross-origin requests, the browser first sends an
        OPTIONS request.

      * This is a permission check.


* Basic CORS headers:

      * Access-Control-Allow-Origin:

            * Defines which origin can access the response.
            * Only one origin or *.
            * Required for all CORS responses.

      * Access-Control-Allow-Credentials:

            * It's allowed to include credentials (cookies, HTTP auth) in this
              cross-origin request and expose the response to javascript.

      * Vary: Origin:

            * Cache-control signal.
            * Without this, caches can serve the wrong CORS headers and break
              valid request.

      * Access-Control-Allow-Methods:

            * Used in preflight response.
            * Tells browser which HTTP methods are allowed.

      * Access-Control-Allow-Headers:

            * Used in preflight response.
            * Allows specific request headers from frontend.

---------- Middlewares ---------------------------------------------------------

* A wrapper that intercepts the request/response pipeline around a handler.


* We use middlewares for various purposes like:

      * Logging
      * Authentication & Authorization
      * Data validation
      * Error handling
      ...


* Basic middleware structure:

      * A middleware is also a handler (http.HandlerFunc type function).

      * But we need to call to the ServeHTTP method of next handler inside a middleware.

      * Therefore we have to wrap the middleware handler with a wrapper function
        to get the next handler as a dependency.

      * Therefore the wrapper function expects a handler and it's return type is
        also a handler.

      * Get a handler (next), return a handler.

      func <middleware_name>(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {

          // Runs before handler

          next.ServeHTTP(w, r)

          // Runs after handler

        })
      }

      * http.Handler interface:

            type Handler interface {
                ServeHTTP(ResponseWriter, *Request)
            }


* To initialize a middleware in the server:

      server := &http.Server{
        ...
        Handler:   <middleware>(mux),
        ...
      }




* Example SecurityHeaders middleware:

      func SecurityHeaders(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
          w.Header().Set("X-DNS-Prefetch-Control", "off")

* Example Compression middleware:

      * To compress the response body.


      * gzWriter := gzip.NewWriter(w)
        defer gzWriter.Close()

      type gzipResponseWriter struct {
        http.ResponseWriter
        gzw *gzip.Writer
      }

      func (g *gzipResponseWriter) Write(b []byte) (int, error) {
        return g.gzw.Write(b)
      }

      func Compression(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
          if !strings.Contains(r.Header.Get("Accept-Encoding"), "gzip") {
            next.ServeHTTP(w, r)
            return
          }

          w.Header().Set("Content-Encoding", "gzip") // required

          gzw := gzip.NewWriter(w)
          defer gzw.Close()

          w = &gzipResponseWriter{
            ResponseWriter: w,
            gzw:            gzw,
          }

          next.ServeHTTP(w, r)
        })
      }




* Example RateLimiter middleware:

      type rateLimiter struct {
        mu            sync.Mutex
        visitors      map[string]int
        limit         int
        resetDuration time.Duration
      }

      func (rl *rateLimiter) reset() {
        for {
          time.Sleep(rl.resetDuration)

          rl.mu.Lock()
          rl.visitors = make(map[string]int)
          rl.mu.Unlock()
        }
      }

      func (rl *rateLimiter) Middleware(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
          visitorIP := r.RemoteAddr

          rl.mu.Lock()
          rl.visitors[visitorIP]++
          visitorCount := rl.visitors[visitorIP]
          rl.mu.Unlock()

          if visitorCount > rl.limit {
            http.Error(w, "Too many requests", http.StatusTooManyRequests)
            return
          }

          next.ServeHTTP(w, r)
        })
      }

      func NewRateLimiter(limit int, resetDuration time.Duration) *rateLimiter {
        rl := &rateLimiter{
          visitors:      make(map[string]int),
          limit:         limit,
          resetDuration: resetDuration,
        }

        go rl.reset()

        return rl
      }

* ExcludeRoutes middleware:

      * To exclude a given middleware to given routes.

      * Do not use "/" at the end for excludedRoutes arguments.

      func ExcludeRoutes(middleware types.Middleware, excludedRoutes ...string) types.Middleware {
        return func(next http.Handler) http.Handler {
          return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            currentRoute := r.URL.Path

            excluded := false

            for _, path := range excludedRoutes {
              if strings.HasPrefix(currentRoute, path) {
                excluded = true
                break
              }
            }

            if excluded {
              next.ServeHTTP(w, r)
            } else {
              middleware(next).ServeHTTP(w, r)
            }
          })
        }

* pool.Exec(...):

            Used for: INSERT, UPDATE, DELETE, CREATE, ETC

* pool.QueryRow(...):

      * Use when you expect exactly one row (or at most one).


      var name string
      var age int

      if err == pgx.ErrNoRows {

* pool.Query(...):

      * Use when you expect multiple rows in the result.


      * Never returns pgx.ErrNoRows.


      students := make([]Student, 0)

      rows, err := pool.Query(context.Background(), "SELECT id,name,age FROM students WHERE age > $1", 50)
      if err != nil {
        http.Error(w, "Failed to fetch data", http.StatusInternalServerError)
        return
      }
      defer rows.Close()

      for rows.Next() {
        var s Student
        err := rows.Scan(&s.ID, &s.Name, &s.Age)
        if err != nil {
          http.Error(w, "Failed to assign data", http.StatusInternalServerError)
          return
        }

        students = append(students, s)
      }

      if rows.Err() != nil {
        http.Error(w, "Failed to fetch data", http.StatusInternalServerError)
        return
      }

      json.NewEncoder(w).Encode(students)


      * pgx.Rows is streamed. Rows are fetched lazily as you call rows.Next().

      * Each call to rows.Next() may fetch more rows from the server.

      * If the connection drops, the server closes the connection, or there's a
        network glitch, rows.Next() will notice it.

      * That's why rows.Err() can return an error after some rows have already
        been processed.




* pool.Acquire(...):

      * To manually get a connection from the pool.


      conn, err := pool.Acquire(ctx)
      defer conn.Release()


      * We can use "pool.AcquireFunc" to auto-manage acquire + release.




* pool.Begin(...):

      * pool.Begin(ctx)    --> starts a transaction
      * pool.Commit(ctx)   --> save changes
      * pool.Rollback(ctx) --> undo changes (ignored if already committed)


      func transfer(ctx context.Context, pool *pgxpool.Pool, from, to, amount int) error {
        tx, err := pool.Begin(ctx)
        if err != nil {
            return err
        }
        defer tx.Rollback(ctx)

        _, err = tx.Exec(ctx,
            "UPDATE accounts SET balance = balance - $1 WHERE id = $2",
            amount, from,
        )
        if err != nil {
            return err
        }

        _, err = tx.Exec(ctx,
            "UPDATE accounts SET balance = balance + $1 WHERE id = $2",
            amount, to,
        )
        if err != nil {
            return err
        }

        return tx.Commit(ctx)
      }


      * Use one context per transaction, Don't mix contexts inside it.


      * Let's say:

            1. Transaction started
            2. First query executed
            3. Internet drops / app crashes
            4. Commit() never happens


      * What PostgreSQL does:

            * PostgreSQL sees client connection disappeared before commit.
            * It will automarically abort the transaction.
            * Changes are not permanent until Commit.
            * They live in a temporary state (WAL + MVCC system).
            * If session ends unexpectedly --> discard changes.


      * We can use pool.BeginTx(...) to start a transaction with custom options.




* pool.CopyFrom(...):

      * Instead of running many INSERT statements (which are slow for large datasets),

---------- Hashing Passwords ---------------------------------------------------

* For hashing passwords it is recommended to use "IDKey" (argon2id) function of
  "argon2" external package.

      golang.org/x/crypto/argon2


* Generate a given number of random bytes:

      salt := make([]byte, 16)  : Generate the byte array with zeros
      _, err := rand.Read(salt) : Replace all zeros with random values (0-255)

      * Never use "math/rand" to generate random bytes. Always use "crypto/rand"


* Salt:

      * A salt is a random value added to a password before hashing.

      * A salt is not a secret.

      * The salt is stored alongside the hash in the database.

      * Why salt is important:

            1. Prevents identical hashes for same password.
            2. Defeats rainbow table attacks.
            3. Slow brute-force attacks.

      * The same salt with the same password will produce the same hash.

      * The commonly recommended salt size is: 16 bytes.


* argon2.IDKey:

      password := "mypassword"
      byteHash := argon2.IDKey([]byte(password), salt, 1, 64*1024, 4, 32)

      * Extra parameters:

            time (iterations):
                Number of passes over memory.

            memory:
                Memory usage in kilobytes.

            threads:
                Number of parallel computation threads.

            keyLen:
                Number of bytes in the derived hash output.


* We don't store raw bytes in most application databases. We convert them to
  base64 strings.


* Format to store hashes in database:

      $<algorithm>$v=<version>$t=<time>$p=<parallelism>$m=<memory>$<base64_salt>$<base64_hash>

      Ex: $argon2id$v=19$t=1$p=4$m=65536$H7aRMAXNz8ZsY/oBwbZNkg==$+kUthIlUfkNuuLVBgK4F770qtZTo6RF3dqN+Qh5fXi0=

      To remember: algo version tpm salt hash (7)


* Ex:

      func main() {
        byteSalt := make([]byte, 16)
        _, err := rand.Read(byteSalt)
        if err != nil {
          log.Fatalln("Failed to generate random bytes")
        }

        password := "mypassword"

        time := 1
        threads := 4
        memory := 64 * 1024

        byteHash := argon2.IDKey([]byte(password), byteSalt, uint32(time), uint32(memory), uint8(threads), 32)
        hash := base64.StdEncoding.EncodeToString(byteHash)
        salt := base64.StdEncoding.EncodeToString(byteSalt)

        data := fmt.Sprintf("$argon2id$v=%d$t=%d$p=%d$m=%d$%s$%s", argon2.Version, time, threads, memory, salt, hash)
        fmt.Println(data)
      }




---------- (Compare Hashed Passwords) ----------

* Step 1: Read encoded string from database


* Step 2: Hash given password with the data in encoded string


* Step 3: Then compare hashes as follows:

        if len(hashedPassword) != len(hash) {
          ...
        }

        if subtle.ConstantTimeCompare(hashedPassword, hash) != 1 {
          ...
        }


* if following parameters are same, argon2id always generates the same hash:

      1. salt
      2. key length
      3. time
      4. threads
      5. memory

---------- Sub Routes ----------------------------------------------------------

* In go standard library (net/http), there is no first-class concept called
  "sub-routes" like in other frameworks.


* Instead, sub-routing is composed behavior built using http.ServeMux,
  handler chaining, and prefix matching.


* What happens on a request?

    func main() {
      childMux := http.NewServeMux()

      childMux.HandleFunc("GET /users", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "users handler")
      })

      mainMux := http.NewServeMux()

      mainMux.Handle("/api/", childMux)

      server := &http.Server{
        Addr:    ":3000",
        Handler: mainMux,
      }

      server.ListenAndServe()
    }


    * Case: request to /api/users.

    * mainMux matches /api/ and forwards the request to childMux.

    * But the request path is still /api/users.

    * childMux tries to match /users

    * No match because it sees /api/users, not /users

    * So your handler is never executed.


* To fix it we can use http.StripPrefix(<prefix>, <handler>):

      It removes a fixed prefix from the request URL path before passing the
      request to another handler.


* fix:

    func main() {
      childMux := http.NewServeMux()

      childMux.HandleFunc("GET /users", func(w http.ResponseWriter, r *http.Request) {
        fmt.Fprint(w, "users handler")
      })

      mainMux := http.NewServeMux()

      mainMux.Handle("/api/", http.StripPrefix("/api", childMux))

      server := &http.Server{
        Addr:    ":3000",
        Handler: mainMux,
      }

      server.ListenAndServe()
    }

        now := time.Now()

        claims := jwt.MapClaims{
          "sub": userID,
          "iat": now.Unix(),
          "exp": now.Add(JWTExpiresIn).Unix(),
        }

        token := jwt.NewWithClaims(jwt.SigningMethodHS256, claims)

        return token.SignedString(JWTSecret)
      }

      func VerifyToken(tokenString string, JWTSecret []byte) (jwt.MapClaims, error) {
        token, err := jwt.Parse(tokenString, func(t *jwt.Token) (any, error) {
          _, ok := t.Method.(*jwt.SigningMethodHMAC)
          if !ok {
            return nil, errors.New("unexpected signing method")
          }

          return JWTSecret, nil
        })
        if err != nil {
          return nil, err
        }

        claims, ok := token.Claims.(jwt.MapClaims)
        if !ok || !token.Valid {
          return nil, errors.New("invalid token")
        }

        return claims, nil
      }

---------- HTTP Polling --------------------------------------------------------

* HTTP Polling is a technique where a client repeatedly sends HTTP requests to a
  server at fixed intervals to check for new data or updates.


* Types of polling:

      1. Short polling:

          Client requests at intervals regardless of server state.


      2. Long polling:

          Client requests, and server holds the request open until new data
          arrives or timeout happens.


Ex (Long Polling):

      func main() {
        mux := http.NewServeMux()

        mux.HandleFunc("GET /long-polling", func(w http.ResponseWriter, r *http.Request) {
          ch := make(chan string)

          go func() {
            time.Sleep(4 * time.Second)

            ch <- "data"
          }()

          select {
          case value := <-ch:
            w.WriteHeader(http.StatusOK)
            fmt.Fprint(w, value)

          case <-time.After(5 * time.Second):
            w.WriteHeader(http.StatusRequestTimeout)
          }
        })

        port := "3000"

        server := http.Server{
          Addr:    fmt.Sprintf(":%s", port),
          Handler: mux,
        }

        fmt.Println("Server is running on port:", port)
        server.ListenAndServe()
      }

---------- Protocol Buffers ----------------------------------------------------

* protocol Buffers (often called protobuf) are a language and platform-neutral
  way of serializing structured data, developed by Google.


* Think of them as a more efficient alternative to formats like JSON or XML for
  sending or storing data.


* Protobuf encodes data into a compact binary format, which makes it:

      * Faster to serialize/deserialize
      * Smaller in size
      * Strongly typed (strict structure)




---------- (Structure Of A .proto File) ----------

* Structure:

      syntax = "proto<version>";

      package <service_name>.<version>;

      import "<another_proto_file>";

      enum <EnumName> {
        <enum_value> = <value_number>;
        ...
      }

      message <MessageName> {
        <field_type> <field_name> = <field_number>;
        ...
      }

      service <ServiceName> {
        rpc <MethodName> (<request_message>) returns (<response_message>);
      }


* syntax:

      Ex: syntax = "proto3";

      * Defines protobuf version.

      * Today almost everything uses: proto3


* package:

      Ex: package auth.v1;

      * Namespace for generated code.

      * Define relatively to the -I option.


* import:

      * To import another .proto file.

      * Do not start with ./

      * When we import a .proto file, we specify path of the file which we want
        to import related to the directory passed via "protoc -I".

      * We use package names of imported files to access their members.

      * You still must import a .proto file even if it uses the same package, as
        long as you want to reference types defined in another file.


* enum:

      Ex:
        enum Status {
          STATUS_UNSPECIFIED = 0;
          STATUS_PENDING = 1;
          STATUS_RUNNING = 2;
          STATUS_COMPLETED = 3;
        }

      * In protobuf enums, values can technically start from any integer, but in
        proto3 the first enum value should be 0.

            * In proto3, enum fields default to 0.

            * If the sender does not set the field, protobuf uses the enum value with number 0.

            * So 0 should represent something like UNSPECIFIED, UNKNOWN, or DEFAULT.


* message:

      Ex:
        message User {
          string id = 1;
          string name = 2;
        }

      * Messages are like structs/classes.

      * Nested messages are allowed.

      * Message fields:

            * By default all fields are optional.

            * Field numbers:

                  * Use positive integers.
                  * Avoid zero.
                  * Unique within a message.
                  * Small numbers: For frequently used fields.
                  * Large numbers: For less common fields.

            * Basic field types:

                  int32, int64, uint32, uint64 (default value: 0)
                  float, double                (default value: 0.0)
                  bool                         (default value: false)
                  string                       (default value: "")
                  bytes                        (default value: empty bytes / [])
                  <enum>                       (default value: default first enum value (usually 0))

            * Special keywords:

                  1. repeated:

                      * To define a list.

                      * Ex:
                            repeated string emails = 2;

                  2. reserved:

                      * To prevent reuse of field numbers and field names.

                      * Ex:
                            message User {
                              reserved 2, 3, 10 to 20;
                              reserved "email", "username";
                            }


* service:

      * To define an RPC interface.

      * It describes methods.




---------- (Serialization / Deserialization) ----------

* Serialization:

      Converting structured data into protobuf binary format.


* Deserialization:

      Converting protobuf binary back into objects/structs.




---------- (RPC) ----------

* RPC: Remote Procedure Call

      It’s a way for one program to ask another program (often on a different
      computer) to run a function or method as if it were local.


* How RPC works:

      1. Client calls a function
      2. RPC system packages the request
      3. Request is sent over the network
      4. Server runs the function
      5. Result is returned to client


* Common RPC technologies:

      * gRPC     - high-performance RPC framework by Google
      * JSON-RPC - lightweight RPC using JSON
      * XML-RPC  - older XML-based RPC
      * SOAP     - enterprise protocol using XML

---------- (Compile .proto Files) ----------

* Before compile, we need to add following line to the .proto file. It
  defines the path & package of generated .pb.go file.

      option go_package = "<your_go_module_name>/<generated_file_relative_path>;<package_name>pb";


* Command:

      protoc -I=proto --go_out=gen --go_opt=paths=source_relative --go-grpc_out=gen --go-grpc_opt=paths=source_relative proto/*/v1/*.proto


      * -I:

          * It tells the compiler where to look for .proto files when resolving
            imports.


      * --go_out=

          * Generated file format: <file_name>.pb.go

          * It includes:
                1. Struct definitions for messages.
                2. Serialization / Deserialization.
                3. Protobuf helpers.


      * --go-grpc_out=

          * Generated file format: <file_name>_grpc.pb.go

          * It includes:
                1. gRPC server interface.
                2. gRPC client stubs.
                3. Service registration helpers.


      * <...>_opt=paths=source_relative

          * Generate output files using the same relative path as the input
            .proto file.

---------- Create A Simple gRPC Server -----------------------------------------

* Folder structure:

      /cmd
          /grpc
              server.go


      /gen


      /internal
          /features
              /<feature1>
                  /v<number>
                      repository.go

                      server.go

                      service.go


      /proto


* If a feature contains multiple services, then define a seperate server file
  for each service.

      <feature1>_server.go
      <feature2>_server.go
      ...


* Examples:

      * internal/features/calculator/v1/repository.go:

            package calculatorv1

            type Repository struct {
              // db *sql.DB
            }

            func NewRepository() *Repository {
              return &Repository{}
            }

            func (repo *Repository) Add(a, b int32) int32 {
              return a + b
            }


      * internal/features/calculator/v1/service.go:

            package calculatorv1

            type Service struct {
              repo *Repository
            }

            func NewService(repo *Repository) *Service {
              return &Service{repo: repo}
            }

            func (service *Service) Add(a, b int32) int32 {
              result := service.repo.Add(a, b)

              // Business rules here

              return result
            }


      * internal/features/calculator/v1/service.go:

            package calculatorv1

            import (
              "context"

              calculatorpbv1 "myproject/gen/calculator/v1"
            )

            type Server struct {
              calculatorpbv1.UnimplementedCalculateServiceServer

              service *Service
            }

            func NewServer(service *Service) *Server {
              return &Server{service: service}
            }

            func (server *Server) Add(ctx context.Context, in *calculatorpbv1.AddRequest) (*calculatorpbv1.AddResponse, error) {
              sum := server.service.Add(in.A, in.B)

              return &calculatorpbv1.AddResponse{Sum: sum}, nil
            }


      * cmd/grpc/server.go:

            func main() {
              const port string = "50051"

              lis, err := net.Listen("tcp", fmt.Sprintf(":%s", port))
              if err != nil {
                log.Fatalln(err)
              }

              grpcServer := grpc.NewServer()

              // calculatorv1
              calculatorV1Repo := calculatorv1.NewRepository()
              calculatorV1Service := calculatorv1.NewService(calculatorV1Repo)
              calculatorV1Server := calculatorv1.NewServer(calculatorV1Service)
              calculatorpbv1.RegisterCalculateServiceServer(grpcServer, calculatorV1Server)

              fmt.Println("Server is running on port:", port)
              err = grpcServer.Serve(lis)
              if err != nil {
                log.Fatalln(err)
              }
            }

---------- Create A Simple gRPC Client -----------------------------------------

* In gRPC, we have to create a seperate client for each service.


* grpc.WithTransportCredentials(insecure.NewCredentials()):

      We have to provide this object to NewClient method if we don't use TLS.


* Ex:

    func main() {
      conn, err := grpc.NewClient("localhost:50051", grpc.WithTransportCredentials(insecure.NewCredentials()))
      if err != nil {
        log.Fatalln(err)
      }
      defer conn.Close()

      client := calculatorpbv1.NewCalculateServiceClient(conn)

      ctx, cancel := context.WithTimeout(context.Background(), time.Second)
      defer cancel()
      req := &calculatorpbv1.AddRequest{A: 10, B: 20}

      res, err := client.Add(ctx, req)
      if err != nil {
        log.Fatalln(err)
      }

      fmt.Println(res.Sum)
    }

* Implement TLS for the server:

      const cert string = "certs/localhost.pem"
      const key string = "certs/localhost-key.pem"

      creds, err := credentials.NewServerTLSFromFile(cert, key)
      if err != nil {
        log.Fatalln(err)
      }

      grpcServer := grpc.NewServer(grpc.Creds(creds))




* Implement TLS for a client:

      const cert string = "certs/localhost.pem"

      creds, err := credentials.NewClientTLSFromFile(cert, "")
      if err != nil {
        log.Fatalln(err)
      }

      conn, err := grpc.NewClient("localhost:50051", grpc.WithTransportCredentials(creds))
