---------- before & after elements ---------------------------------------------

* To add some small content to the current element like labels.


* Always you must specify the elements as follows. Otherwise styles are not
  applied as expected:

    className="before:absolute before:content-['']"

    className="after:absolute after:content-['']"

---------- z index -------------------------------------------------------------

* To control the stacking order in z axis.


* To apply the z index, the element must be one of followings:

    * relative
    * absolute
    * fixed


* If you don't need a specific positioning, just use: relative


* Send an item to the bottom of the stack:

    -z-<value>


* Send an item to the top of the stack:

    z-<value>

--------------------------------------------------------------------------------




---------- Grid ----------------------------------------------------------------

* Default:

    * width: Acquire the width as much as possible.

    * height: Based on the content.

    * grid columns = 1


*  The grid is devided into grid cells by horizontal & verical grid lines.


* There is a special way to specify the positions of these grid lines.

    Horizontal:
      left to right : 1, 2, 3, 4, ...
      right to left : -1, -2, -3, -4, ...

    Vertical:
      top to bottom : 1, 2, 3, 4, ...
      bottom to top : -1, -2, -3, -4, ...


* The most outer html element of each grid cell acquires full width and height
  of the cell. (size-full)


* The height of a grid row is bases on the grid cell which has the maximum height.




---------- Container level classes ----------

* grid : convert a html element to a grid.


* grid-cols-2: Specify the number of columns of the grid. (default 1)


* gap-x-2 : Horizontal gap between grid cells.


* gap-y-2 : Vertical gap between grid cells.


* gap-2 : Both Horizontal & vertical gap.


* Align items in each grid cell:

    * Horizontally: justify-items-<position> (Default: justify-items-stretch)

    * Vertically  : items-<position> (Default: items-start)




---------- Cell level classes ----------

* To expand a cell in 2 columns including current one: col-span-2

* To expand a cell in 2 rows including current one   : row-span-2

* When to use each <h/> tags:

    h1: Page’s main title.

    h2: Main sections under the H1.

    h3: Subsections under H2.

    h4: Sub-subsections under H3 (rare)

    h5: Deeply nested content (very rare)

    h6: Lowest-level nested content (rare)

---------- States --------------------------------------------------------------

---------- Data attribute ----------

* Syntax: data-<state_name>=<string_value>

    Ex: data-state="open"


* Normal usage:

<div
  data-color="green"
  className="bg-amber-500 data-[color=green]:bg-emerald-500 w-96 aspect-video rounded-md"
></div>




* With peer:

<div className="min-h-screen flex gap-4 justify-center items-center">
  <div
    data-color="amber"
    className="peer bg-amber-500 w-96 aspect-video rounded-md"
  ></div>

  <div className="bg-slate-500 w-96 peer-data-[color=amber]:bg-amber-500 aspect-video rounded-md"></div>
</div>




* With peer + custom name:

<div className="min-h-screen flex gap-4 justify-center items-center">
  <div
    data-color="amber"
    className="peer/first bg-amber-500 w-96 aspect-video rounded-md"
  ></div>

  <div className="bg-slate-500 w-96 peer-data-[color=amber]/first:bg-amber-500 aspect-video rounded-md"></div>
</div>




* With group:

<div
  data-color="amber"
  className="group bg-slate-500 w-96 aspect-video rounded-md flex justify-center items-center"
>
  <div className="bg-amber-500 group-data-[color=amber]:bg-emerald-500 w-80 aspect-video rounded-md"></div>
</div>




* With group + custom name:

<div
  data-color="amber"
  className="group/parent bg-slate-500 w-96 aspect-video rounded-md flex justify-center items-center"
>
  <div className="bg-amber-500 group-data-[color=amber]/parent:bg-emerald-500 w-80 aspect-video rounded-md"></div>
</div>




* An example with useState:

export default function HomePage() {
  const [state, setState] = useState<"close" | "open">("close");

  function handleState() {
    setState(state == "open" ? "close" : "open");
  }

  return (
    <div className="min-h-screen flex gap-4 justify-center items-center">
      <Button onClick={handleState}>Toggle</Button>

      <ChevronDown
        data-state={state}
        className="data-[state=close]:rotate-180 transition-transform"
      />
    </div>
  );
}

Note: We use "transition-transform" tailwind class for smooth transforms.

---------- Container Queries ---------------------------------------------------

* For styling based on parent container(s) width. (Not based on screen width)


* Pre-defined container sizes:

    xs  ----------> 20rem
    sm  ----------> 24rem
    md  ----------> 28rem
    lg  ----------> 32rem
    xl  ----------> 36rem
    2xl ----------> 42rem
    3xl ----------> 48rem
    4xl ----------> 56rem
    5xl ----------> 64rem
    6xl ----------> 72rem
    7xl ----------> 80rem




* Based on one parent container (@container):

<div className="@container">
  <div className="flex flex-col @md:flex-row">
    // Content
  </div>
</div>




* Based on nested parent containers (@container/<name>):

<div className="@container/main">
  <div className="flex flex-col @md/main:flex-row">
    // Content
  </div>
</div>




* Create reusable custom container sized:

    @theme {
      --container-<container_size_name>: <width>;
    }




* Using arbitrary values:

    @min-[475px]
    @max-[475px]

    <div className="@container">
      <div className="flex flex-col @min-[475px]:flex-row">
        // Content
      </div>
    </div>
