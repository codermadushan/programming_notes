---------- DashLine to place inside Stack --------------------------------------

Ex:

class DashLine extends StatelessWidget {
  final double _dashHeight; // required
  final double _dashWidth;
  final Color _dashColor;
  final double _paddingHorizontal;

  const DashLine({
    super.key,
    required Color dashColor,
    required double dashHeight,
    double dashWidth = 6.0,

    double paddingHorizontal = 0,
  }) : _dashColor = dashColor,
       _dashWidth = dashWidth,
       _dashHeight = dashHeight,

       _paddingHorizontal = paddingHorizontal;

  @override
  Widget build(BuildContext context) {
    return Padding(
      padding: EdgeInsets.symmetric(horizontal: _paddingHorizontal),
      child: LayoutBuilder(
        builder: (context, constraints) {
          return Row(
            mainAxisAlignment: MainAxisAlignment.spaceEvenly,

            children: List.generate(
              (constraints.maxWidth - _dashWidth) ~/ (2 * _dashWidth),
              (index) {
                return ColoredBox(
                  color: _dashColor,
                  child: SizedBox(width: _dashWidth, height: _dashHeight),
                );
              },
            ),
          );
        },
      ),
    );
  }
}


Note:

    * "dashHeight" should be a required parameter. Because when you position this
      inside stack you need to access to dashHeight.

    * If you use it inside a Stack widget, then wrap it with Sizedbox and give a
      fixed width. Otherwise you will get an error.

    * If you want to use it inside a Row, then wrap it with Expanded widget.

    * Main Structure:

        Padding
          LayoutBuilder
            Row
              ColoredBox
                Sizedbox

---------- "late" keyword ------------------------------------------------------

* We use late keyword for lazy initialization in dart.




---------- (Lazy initialization to a variable) ----------

* In dart if you create a non-nullable variable outside of a function or method,
  you must provide a value for that variable.

      String name; ❌

      String name = 'Madushan'; ✔️


* But using "late" keyword we can create a non-nullable variable outside of a
  method without initializing a value. But we must provide a value to it before
  access. Otherwise it will throw an exception.

      late String name; ✔️


* We can define a non-nullable variable with "late final" keywords. In this case
  the variable can be assigned only once.

    late final String name;

    void main() {
      name = 'Madushan'; ✔️
      print(name);

      name = 'Madushan'; ❌
    }


* We must use "late" before "final".

    late final ✔️

    final late ❌




---------- (Lazy initialization to a function) ----------

* When we get a value from a function we can use late keyword. In this case
  the function is executed only when we access the returned value.


Ex:

String provideName() {
  print('"provideName" executed');

  return 'Madushan';
}

void main() {
  print('Start');

  late String name = provideName();

  print('End');

  print(name);
}


* Result:

    Start
    End
    "provideName" executed
    Madushan


* Even if we access the returned value multiple times, function is called only
  once and returned value is stored in the late viriable and it is reused.




---------- (Lazy initialization inside a class)  ----------

* In dart you cannot use a instance variable to initialize another instance
  variable.

Ex:

class Person {
  final String firstName;
  final String lastName;
  final String fullName = '$firstName $lastName'; ❌

  Person({required this.firstName, required this.lastName});
}


* But you can do it using "late" keyword.

Ex:

class Person {
  final String firstName;
  final String lastName;
  late final String fullName = '$firstName $lastName'; ✔️

  Person({required this.firstName, required this.lastName});
}

---------- Mixins --------------------------------------------------------------

* Mixins are seperate entities that can bring functionalities to existing
  classes and enums.


* Mixins are used to share behavior, not enforce structure. Therefore in general
  following entities inside a mixin.

      1. Concrete methods.

      2. Abstract instance variables (getters and setters) or abstract getters
         if we need to use it inside a concrete method of current mixin or
         inside a mixin which is built on current mixin.


* If you build a new mixin on another mixin (using "on" keyword), we can access
  abstract instance variable in parent mixin in child mixin.

      The reason is:

          1. You must provide implementation for abstract instance variable
             (or getter and setter) in mixed class (using "with" keyword) and
             all concrete methods in all mixins available as instance methods
             in mixed class.

          2. If you want to build a mixin on another mixin and mix it with a
             class you have to mix the class with both child and parent mixin.


Ex:

mixin HasSpeed {
  int get speed;
}

mixin CanJump on HasSpeed {
  void jump() {
    print('$runtimeType is jumping at the speed of: $speed');
  }
}

mixin CanWalk on HasSpeed {
  void walk() {
    print('$runtimeType is walking at the speed of: $speed');
  }
}

class Person with HasSpeed, CanJump, CanWalk {
  final int _speed;

  const Person(int speed) : _speed = speed;

  @override
  int get speed => _speed;
}

void main() {
  final p1 = Person(10);
  final p2 = Person(5);

  p1.jump();
  p1.walk();

  p2.jump();
  p2.walk();
}

import 'package:flutter/material.dart';

class TypeWriter extends StatelessWidget {
  static const _defaultCharDurationMs = 100;
  static const _defaultCursorChar = '_';

  final String _text;
  final String _cursorCharacter;
  final int _durationMs;
  final TextStyle? _textStyle;
  final Curve? _curve;

  const TypeWriter({
    super.key,
    required String text,
    String cursorCharacter = _defaultCursorChar,
    int? durationMs,
    TextStyle? textStyle,
    Curve? curve,
  })  : _text = text,
        _cursorCharacter = cursorCharacter,
        _durationMs = durationMs ?? text.length * _defaultCharDurationMs,
        _textStyle = textStyle,
        _curve = curve;

  @override
  Widget build(BuildContext context) {
    return TweenAnimationBuilder(
      tween: IntTween(begin: 0, end: _text.length),
      duration: Duration(milliseconds: _durationMs),
      curve: _curve ?? Curves.linear,
      builder: (context, value, child) {
        final current = _text.substring(0, value);
        final isCompleted = _text.length == value;
        return Text(
          isCompleted ? current : '$current$_cursorCharacter',
          style: _textStyle,
        );
      },
    );
  }
}

---------- (Skeleton) ----------

class Skeleton extends StatefulWidget {
  final Color _color;
  final double _borderRadius;
  final double? _width;
  final double? _height;

  const Skeleton({
    super.key,
    Color color = Colors.grey,
    double borderRadius = 10.4,
    double? width,
    double? height,
  }) : _color = color,
       _borderRadius = borderRadius,
       _width = width,
       _height = height;

  @override
  State<Skeleton> createState() => _SkeletonState();
}

class _SkeletonState extends State<Skeleton> {
  late Color _begin;
  late Color _end;

  @override
  void initState() {
    super.initState();

    _begin = widget._color;
    _end = widget._color.withAlpha(128);
  }

  @override
  Widget build(BuildContext context) {
    return TweenAnimationBuilder(
      duration: const Duration(seconds: 1),
      tween: ColorTween(begin: _begin, end: _end),

      onEnd: () {
        setState(() {
          final temp = _begin;
          _begin = _end;
          _end = temp;
        });
      },

      builder: (context, value, child) {
        return Container(
          width: widget._width,
          height: widget._height,

          decoration: BoxDecoration(
            color: value,
            borderRadius: BorderRadius.circular(widget._borderRadius),
          ),
        );
      },
    );
  }
}

---------- covariant keyword ---------------------------------------------------

* Used to loosen type safety in method parameters so that a subclass can override
  a method and accept a more specific type than the superclass.


* Normally, Dart requires that overridden methods must accept the same or more
  general parameter types. But in some cases, you want a subclass to accept a
  more specific type.


Ex:

    class Animal {}

    class Dog extends Animal {}

    class AnimalHandler {
      void handle(Animal animal) {
        //
      }
    }

    class DogHandler extends AnimalHandler {
      @override
      void handle(Dog dog) { // error
        //
      }
    }


    * In above example when we override the handle method of AnimalHandler class
      in DogHandler with a Dog parameter type, we get an error. Because you
      break the normal type rules.


    * We can fix this as follows using "covariant" keyword.

          class Animal {}

          class Dog extends Animal {}

          class AnimalHandler {
            void handle(covariant Animal animal) {
              //
            }
          }

          class DogHandler extends AnimalHandler {
            @override
            void handle(Dog dog) {
              //
            }
          }

---------- (Transform) ---------------------------------------------------------

Transform(
  alignment: Alignment.center,
  transform: Matrix4.identity()..rotateZ(pi / 2),
  child: <Widget>,
)


* Properties:

      * alignment:
          * The pivot point of the transform.

          * For custom pivot points, use "origin" property instead of alignment.
              origin: const Offset(X, Y),


      * transform:
          * To specify which kind of transform you want.

          * Matrix4.identity() is something like no transform.


* Always remember that when you rotate a widget, the canvas is also rotated

  axis are changed.


* To translate a widget along the Z axis, we have to use the package called
  "vector_math" as follows.

      1. import 'package:vector_math/vector_math_64.dart' show Vector3;

      2. Matrix4.identity()..translate(Vector3(X, Y, Z))

---------- Clip widgets --------------------------------------------------------

---------- (Path object) ----------

* Create path object (It is mutable):
    final path = Path();


* We use path to mark the path of the area which we want to keep.


* Methods:

      * path.moveTo(X, Y):
            Starting point.
            Default: path.moveTo(0, 0) -> top-left corner


      * path.lineTo(X, Y):
            Draws a straight line to this point


      * path.arcToPoint(
          offset,
          radius: ,
          clockwise: ,
        ):
            To draw an arc


      * Finally draw a straight line from last end point to start point using:
            path.close();




---------- (CustomClipper) ----------

class <ClassName> extends CustomClipper<Path> {
  @override
  Path getClip(Size size) {
    //
  }

  @override
  bool shouldReclip(covariant CustomClipper<Path> oldClipper) {
    //
  }
}


* We mostly use CustomClipper class with "Path" object. But we can use "Rect" or
  "RRect" objects if we need.


* "getClip" method provides the size of the widget which we are going to clip
  automatically using on of following widget,

    * ClipPath
    * ClipRect
    * ClipRRect
    * ClipOval

  and we need to return a suitable object in this method which is compatible with
  CustomClipper generic type.


* The "shouldReclip" method tells Flutter whether it needs to recalculate the
  clipping path when the clipper changes.
    * If your widget is static, return false.
    * If the widget size can be changed, return true.


* To apply custom clipper to the widget use one of following class and provide
  CustomClipper object to them using clipper property.
    * ClipPath
    * ClipRect
    * ClipRRect
    * ClipOval


Ex: Creating a circle using 2 squares in different colors,

    class _CircleClipper extends CustomClipper<Path> {
      @override
      Path getClip(Size size) {
        final radius = Radius.circular(size.height / 2);

        return Path()
          ..moveTo(size.width / 2, 0)
          ..arcToPoint(Offset(size.width / 2, size.height), radius: radius)
          ..arcToPoint(Offset(size.width / 2, 0), radius: radius);
      }

      @override
      bool shouldReclip(covariant CustomClipper<Path> oldClipper) => false;
    }

    class Circle extends StatelessWidget {
      const Circle({super.key});

      @override
      Widget build(BuildContext context) {
        const size = 200.0;

        return ClipPath(
          clipper: _CircleClipper(),
          child: Row(
            mainAxisSize: .min,
            children: [
              Container(width: size, height: size, color: Colors.blue),

* To draw any kind of shape on a canvas.

final class Polygon extends CustomPainter {
  final int _sides;

  const Polygon(int sides) : _sides = sides;

  @override
  void paint(Canvas canvas, Size size) {
    final paint = Paint()
      ..color = Colors.blue
      ..style = PaintingStyle.stroke
      ..strokeCap = StrokeCap.round
      ..strokeWidth = 3;

    final center = Offset(size.width / 2, size.height / 2);

    final radius = size.width / 2;

    final angle = 2 * pi / _sides;
    final angles = List.generate(_sides, (index) => index * angle);

    final path = Path();
    path.moveTo(center.dx + radius * cos(0), center.dy + radius * sin(0));
    for (final angle in angles) {
      if (angle == 0) continue;
      final x = center.dx + (radius * cos(angle));
      final y = center.dy + (radius * sin(angle));
      path.lineTo(x, y);
    }
    path.close();

    canvas.drawPath(path, paint);
  }

  @override
  bool shouldRepaint(covariant CustomPainter oldDelegate) {
    return oldDelegate is Polygon && oldDelegate._sides != _sides;
  }
}

---------- (AnimationController) ----------

* Moves from double lowerBound to double upperBound in given duration.


* Constructor properties:

      vsync:

          * A TikerProvider to prevent unnecessary animation work
          * Usually mix the widget with SingleTickerProviderStateMixin or
            TickerProviderStateMixin and provide "this" to the vsync.


      value:

          * To set the initial value of the controller
          * Default: lowerBound


      lowerBound:

          * Minimum value the controller can reach
          * Default: 0.0


      upperBound:

          * Maximum value the controller can reach
          * Default: 1.0


      duration:

          * Duration for forward animation
          * Default: null
          * If not provided, you must set it manually before calling forward.
            Otherwise An error is thrown.


      reverseDuration:

          * Duration for reverse
          * Default: uses "duration"


      animationBehavior:

          * You almost never need to change this.
          * Controls the controller's behavior in low-power mode
          * Two possible values:

                1. AnimationBehavior.preserve (default):

                      Animation plays normally even when "reduce motion" is ON


                2. AnimationBehavior.normal:

                      Linearizes animation. (removes curves, shortcuts long animations)


      debugLabel:

          * Label used in Flutter DevTools/debugging.
          * Helps you identify controllers.
          * Does not effect performance or behavior.




* Does not run automatically.


* "lowerBound" <= "upperBound"


* Cannot provide curves directly.


* To control the animation we always use AnimationController even there is an
  Animation instance,


* Methods:

      _animationController.forward():

          * Runs the animation forward.
          * _animationController.value is always linear.
          * Can await until animation completes.
          * Properties:

                from:
                    * Jump to a specific start value before running.
                    * Default: lowerBound.


      _animationController.reverse():

          * Runs the animation backward.
          * _animationController.value is always linear.
          * Can await until animation completes.
          * Properties:

                from:
                    * Jump to a specific start value before running.
                    * Default: upperBound.


      _animationController.repeat():

          * Loops the animation continuously from lowerBound to upperBound.
            After completed, then start again in lowerBound.

          * _animationController.value is always linear.

          * Properties:

                * min:
                      Start value of the repeating range.
                      min >= lowerBound
                      Default: lowerBound

                * max:
                      End value of the repeating range.
                      max <= upperBound
                      Default: upperBound

                * reverse:
                      Default: false

                      If true animation runs from lowerBound to upperBound and
                      then upperBound to lowerBound. Then continue the process
                      again and again.

                * period:
                      Default: _animationController's duration/reverseDuration.

                      Replace the _animationController's duration for that
                      specific repeating cycle.

                * count:
                      Default: repeats indefinitely.

                      Specify how many times the animation should repeat.


      _animationController.animateTo(<target>):

          * Animates from current value -> target.
          * Forward if target > current value.
          * Backward if target < current value.

          * _animationController.value is not linear if you don't use,
                curve: Curves.linear

          * Target must be within controller's bounds.
          * It supports curves.
          * Properties:

                * target:
                      Target value to animate from current value.

                * duration:
                      Duration for the animation.
                      Default: Controller's default duration.

                * curve:
                      To specify the curve.
                      Default: Curves.linear


      _animationController.animateBack(<target>):

          * Simillar to animateTo, but always backward.


      _animationController.reset():

          * Sets the controller's value back to lowerBound.


      _animationController.stop():

          * To stop the animation immediately.
          * Properties:

                * canceled: (default: true)

                      true:
                          Animation is interrupted.
                          No status change happens.

                      false:
                          Treat stop as natural finish.

                          If the controller was moving forward: trigger
                          AnimationStatus.completed

                          If the controller was moving backward: trigger
                          AnimationStatus.dismissed


      _animationController.dispose():

          * Release resources when the controller is no longer needed.




* Properties:

      _animationController.isAnimating:
          Animation is running forward or backward.

      _animationController.isCompleted:
          Animation is stopped at the end.

      _animationController.isDismissed:
          Animation is stopped at the beginning.

      _animationController.isForwardOrCompleted:
          Animation is running forward or stopped at the end.

      _animationController.value:
          Directly access the controller's current value.

      _animationController.value = x:
          Directly set the controller's value.




* If you use forward method, first time animation will be executed. But then if
  you hot reload again, animation will not be executed. Because it already in
  forwarded position. In this case it is important to always use reset() method before
  the forward() method

    * animationController..reset()..forward();




* Register/Remove a callback function that runs every time the animation's value
  changes.

      void _listener() {
        //
      }


      To register:
          _animationController.addListener(_listener)

      If need to remove before dispose:
          _animationController.removeListener(_listener)




* We can also register/remove a listener to listen animation status as follows,

      void _statusListener(AnimationStatus status) {
        if (status == AnimationStatus.completed) {
          //
        }
      }


      To register:
          _animationController.addStatusListener(_statusListener)

      If need to remove before dispose:
          _animationController.removeStatusListener(_statusListener)


      AnimationStatus:

          * completed:
                Animation is stopped at the end.

          * dismissed:
                Animation is stopped at the beginning.

          * forward:
                Animation is running forward.

          * reverse:
                Animation is running backward.

        class TodosCubit extends Cubit<TodosState> {
          static const _limit = 20;
          static int _page = 1;

          final LoadTodos _loadTodos;

          TodosCubit({required LoadTodos loadTodos})
            : _loadTodos = loadTodos,
              super(const TodosInitial());

          Future<void> loadTodos() async {
            if (state is TodosLoading) return; // This line is very important

            emit(TodosLoading(todos: state.todos, isFirstLoading: _page == 1));

            final params = LoadTodosType(limit: _limit, page: _page);
            final res = await _loadTodos(params);

            res.fold(
              (failure) {
                emit(TodosError(todos: state.todos, message: failure.message));
              },
              (todos) {
                emit(TodosSuccess(todos: state.todos + todos));
                _page++;
              },
            );
          }
        }

            itemBuilder: (_, index) {
              if (index < todos.length) {
                return TodoCard(todos[index]);
              }

              // To make sure that page is scrolled up after rendering loading indicator below.
              Future.delayed(const Duration(milliseconds: 1), () {
                _scrollController.jumpTo(
                  _scrollController.position.maxScrollExtent,
                );
              });

              return const Padding(
                padding: EdgeInsetsGeometry.symmetric(vertical: 8),
                child: Center(child: CircularProgressIndicator()),

---------- Classes -------------------------------------------------------------

* We use classes to represent real world entities in the programs.


* A class contains following members:

    Instance members (object level):
        * Instance variables:
        * Instance methods

    Static members (class level):
        * static variables
        * static methods


* When we should use instance variables?
    If each object of the class should contains some property, then make it a
    instance variable. (like name, age in Student class. Because every student
    object has it's own name and age)

    Inherited to sub classes.


* When we should use instance methods?
    If we want to access instance members inside a method, then make it a
    instance method.

    To call to a instance method, we have to use an object.
        <object>.<method>() {}

    Inside the instance method, we can access current object using "this"
    keyword.

    Inherited to sub classes.


* When we should use static variables?
    Static variables are not related to specific object. They are class level
    variables. Can call only using class name.

    If you want to create variables that are not related to objects but related
    to the class, then make it a static variable.

    Not inherited to sub classes.


* When we should use static methods?
    In a method, if you don't use any instance variable or instance method, then
    make it a static method. To call to a static method, we have to use the
    class name.

    Not inherited to sub classes.


* Common best practices when creating a class:

    1. When we define instance variables, make them,
        * final
        * private

    2. To access, modify instance variables outside of the class use,
        * getters
        * setters

        This is called encapsulation.




---------- (final classes) ----------

* Once you mark a class as final, no other library can extend or implement it.

Ex: final class Person {}

--------------------------------------------------------------------------------




---------- Create a model class ------------------------------------------------

* By this model class we can,

    1. Send data to database.
    2. Get data from database.


* For a model class we define all constructor arguments as named arguments.


* There are 4 types of instance variable fields,

    1. Named required.
    2. Named optional with const default value.
    3. Named optional with non const default value.
    4. Named optional nullable.


Ex:

@immutable
class UserEntity {
  final String name;        // as non nullable

  final bool isActive;      // as non nullable

  final DateTime createdAt; // as non nullable

  final String? email;      // as nullable


  UserEntity({
    required this.name,   // named required

    this.isActive = true, // named optional with const default value

    DateTime? createdAt,  // named optional with non const default value

    this.email,           // named optional nullable
  }) : createdAt = createdAt ?? DateTime.now();

  Map<String, dynamic> toMap() {
    return {
      'name': name,
      'is_active': isActive,
      'created_at': createdAt.toIso8601String(),
      'email': email,
    };
  }

  factory UserEntity.fromMap(Map<String, dynamic> map) {
    return UserEntity(
      name: map['name'],
      isActive: map['is_active'],
      createdAt: DateTime.parse(map['created_at']),
      email: map['email'],
    );
  }
}


* toMap is a instance method. Inherited to sub classes.

* fromMap is a factory constructor. Not inherited to sub classes.

---------- extends vs implements -----------------------------------------------

* extends:

    Used to inherite from a normal class (concrete class) or abstract class.

    Need to provide implementation for all abstract instance methods. For concrete
    methods implementation is optional.

    Parent class concrete instance methods are available in subclasses.


* implements:

    Used to inherite from an interface.

    Need to provide implementation for all methods.
    (abstract methods + concrete methods)

--------------------------------------------------------------------------------




---------- Inheritance ---------------------------------------------------------

* By using inheritance we can inherite parent class instance variables and
  instance methods to child classes.


Ex:

class Person {
  final String _name;
  final int _age;

  const Person({required String name, required int age})
    : _name = name,
      _age = age;

  void walking() {
    print('Person is walking');
  }

  void showDetails() {
    print('Name  : $_name');
    print('Age   : $_age');
  }
}

class Employee extends Person {
  final int _salary;

  const Employee({required String name, required int age, required int salary})
    : _salary = salary,
      super(name: name, age: age);

  @override
  void walking() {
    print('Employee is walking');
  }

  @override
  void showDetails() {
    super.showDetails(); // Method body of parent class showDetails method.
    print('Salary: $_salary');
  }
}


1. In child class constructor you should get values for all instance variables
   including parent class instance variables and initialize inherited instance
   variables from parent class using super().

    Ex:

      const Employee({required String name, required int age, required int salary})
        : _salary = salary,
          super(name: name, age: age);

    Now you can imagine all parent class instance variables available in sub
    class.


2. You can completely override a method which is inherited from parent class
   inside sub class like "walking" method in above example.


3. Instead of completely overriding a method which is inherited from parent
   class, we can override it extending parent class implementation also like
   "showDetails" method in above example.

    Ex:

      @override
      void showDetails() {
        super.showDetails();
        print('Salary: $_salary');
      }

    "super.showDetails()" means the method body of parent class "showDetails"
    method.

---------- Abstract classes ----------------------------------------------------

* We use abstract classes for:

    1. Force sub classes to provide implementations for specified instance
       methods (use abstract instance methods).

    2. Provide concrete instance methods for sub classes.

    3. Provide common instance variables to sub classes.


* We cannot create objects for an abstract class.


* Always use "extends" for inherite to a sub class.


* Mainly it contains following entities:

    1. instance variables:

        * That are common for extended sub classes.


    2. Concrete instance methods:

        * An instance methods with the body.

        * Automatically available for sub classes.

        * Can provide implementation in sub classes (override). But it is optional.


    3. Abstract instance methods

        * An instance method without the body.

        * Must provide implementation in sub classes (override) or we have to
          make the sub class an abstract class.


* If we cannot create objects of an abstract class, why constructors are allowed?
    To initialize it's instance variable in sub classes using super().


* Can contains static variables:
    Should be initialize at the time of creation or Should be nullable.

    Not inherited to sub classes.


* Can contains static methods:
    Method body is required.

    Not inherited to sub classes.


Ex:

abstract class Person {
  // Instance variables that are common for sub classes
  final String _name;

  const Person(String name) : _name = name;

  // Method with the body (concrete method)
  void eating() {
    print('Person is eating');
  }

  // Method without the body (abstract method)
  void walking();
}

class Student extends Person {
  final String _country;

  Student({required String name, required String country})
    : _country = country,
      super(name);

  @override
  void walking() {
    print('Student is walking');
  }
}




---------- (sealed classes) ----------

* Same as abstract classes. But can't be extended, or mixed (with mixin) in
  outside of its library (current .dart file)

* Simply called "Implicitly abstract".

--------------------------------------------------------------------------------




---------- Interfaces ----------------------------------------------------------

* We use interfaces for:

    1. Force sub classes to provide implementations for specified getters.

          <return_type> get <method_name>;


    2. Force sub classes to provide implementations for specified instance
       variables (or getters and setters).

          abstract <data_type> <getter_or_setter_name>;


    3. Force sub classes to provide implementations for specified instance
       methods (use abstract instance methods).

          <return_type> <method_name>(args);


* They are 100% abstract classes. No concrete instance methods.


* Objects are not allowed for interfaces.


* Constructors are not meaningful for interfaces.


* Always use "implements" for inherite to sub classes.


* Can contains static variables:
    Should be initialize at the time of creation or Should be nullable.

    Not inherited to sub classes.


* Can contains static methods:
    Method body is required.

    Not inherited to sub classes.


Ex:

abstract interface class Person {
  // For abstract getters
  String get name;

  // For abstract getter & setters
  abstract int age;

  // abstract methods
  void walking();
}

class Student implements Person {
  final String _name;
  int _age;

  Student({required String name, required int age}) : _name = name, _age = age;

  @override
  String get name => _name;

  @override
  int get age => _age;

  @override
  void set age(int age) => _age = age;

  @override
  void walking() {
    print('Student is walking');
  }
}

* domain/entity:

    To define models to use inside presentation layer.

    from database. Therefore when we design a entity model class, we design it

    need to consider only 2 kind of arguments for the model.

        1. Named required.
        2. Named optional nullable.

              "Named optional with non const default value" only if we send data
              to database via the model.

* data/datasource:

    To define,

        1. The interface for data sources implementations.
        2. Data sources implementations.

    Ex:

      1. Interface:

        Naming convension: <feature_name>DataSource

        abstract interface class TodosDataSource {
          Future<List<TodoModel>> loadTodos({required int limit, required int page});
        }


      2. Implementations:

        Naming convension: <feature_name>RemoteDataSource or <feature_name>LocalDataSource

        class TodosRemoteDataSource implements TodosDataSource {
          static const _baseUrl = String.fromEnvironment(AppString.baseUrl);

          const TodosRemoteDataSource();

          @override
          Future<List<TodoModel>> loadTodos({
            required int limit,
            required int page,
          }) async {
            try {
              final start = (page - 1) * limit;

              final res = await http.get(
                Uri.parse('$_baseUrl/todos?_limit=$limit&_start=$start'),
                headers: {'Content-Type': 'application/json'},
              );

              if (res.statusCode != 200) {
                throw const ServerException('Invalid response');
              }

              final todos = jsonDecode(res.body) as List<dynamic>;
              final todoModels = todos.map((todo) => TodoModel.fromMap(todo)).toList();

              print(todoModels[0].toMap());

              return todoModels;
            } on ServerException catch (e) {
              throw ServerException(e.message);
            } catch (e) {
              throw const ServerException(
                'Please check your internet connection or try again later',
              );
            }
          }
        }


        * If you have nothing to return from a function, return "unit" comes
          with "fpdart" package instead of void. The data type of "unit" is "Unit"




* data/repository:

    Implements from domain/repository.

    We perform following operations here,

        1. Condition checks
        2. Logics
        ...

    In here we substitute model class with entity class.

    In here in each data source repository implementation, we get the
    <feature_name>DataSource interface as a dependency.

    Ex:

        class TodosRemoteRepository implements TodosRepository {
          final TodosDataSource _todosDataSource;

          const TodosRemoteRepository(TodosDataSource todosDataSource)
            : _todosDataSource = todosDataSource;

          @override
          Future<Either<Failure, List<TodoEntity>>> loadTodos({
            required int limit,
            required int page,
          }) async {
            try {
              final todoModels = await _todosDataSource.loadTodos(
                limit: limit,
                page: page,
              );
              return right(todoModels);
            } on ServerException catch (e) {
              return left(Failure(e.message));
            }
          }
        }


* domain/usecase:

    We should define usecases for each function declarations in
    <feature_name>Repository interface.


    We should implements each usecase from one of following interface (in lib/core/usecases).

        1. AsyncUseCase: for asynchronous operations.

            Ex:
                abstract interface class AsyncUseCase<SuccessType, ParamsType> {
                  Future<Either<Failure, SuccessType>> call(ParamsType params);
                }

        2. SyncUseCase: for synchronous operations.

        Note:
          You can create more usecase interfaces for your needs.


    We should get <feature_name>Repository as a dependency into each usecase.

    Ex:

        @immutable
        class LoadTodosParam {
          final int limit;
          final int page;

          const LoadTodosParam({required this.limit, required this.page});
        }

        class LoadTodos implements AsyncUseCase<List<TodoEntity>, LoadTodosParam> {
          final TodosRepository _todosRepository;

          const LoadTodos(TodosRepository todosRepository)
            : _todosRepository = todosRepository;

          @override
          Future<Either<Failure, List<TodoEntity>>> call(LoadTodosParam param) async {
            return await _todosRepository.loadTodos(
              limit: param.limit,
              page: param.page,
            );
          }
        }


* presentation/bloc:

    We define the state/states of the feature here.

        Use seperate cubits for each value. But if there is only one value for
        the feature like auth (only User), then you can define the single cubit
        using feature name (AuthBloc).

        Naming convension:

            Inside the feature:
                If there is only one value for the feature:
                    <feature_name>Bloc

                If there are multiple values for the feature:
                    <value_name>Bloc

            If need to use appwide:
            Always: App<value_name>Bloc


    We get usecases in domain/usecase as dependencies to the blocs in
    presentation/bloc.


    If you need to use a state outside of the feature which it is defined, create a
    new state in "lib/core/bloc/<cubit_name>" and get it as a dependency
    into the feature cubit.


    As an example, most of the time you need to access user infomations more than
    one features. Therefore create a new cubit called "app_user" (always prefix
    name of the bloc with "app_") in "lib/core/bloc" and get it as a dependency
    into the auth_bloc in auth feature. Then you can update app_user state inside
    auth_cubit.

        In this case if you want to call a method in AuthBloc using AppUserBloc,
        then define a instance variable as follow in AppUserBloc

            late final AsyncCallback <public_method_name>;

        Now inside AuthBloc constructor body you can initialize it as follows,

            _appUserBloc.<public_method_name> = <name_of_the_method_you_want_to_call>;

        If you get any uninitialized error when you call the method using AppUserBloc
        the reason is AuthBloc is not initialized yet. In this case you can
        use registerSingleton and pass the AuthBloc directly to it without using
        callbacks as follows,

            registerSingleton(AuthBloc(...)); // correct

            registerSingleton(() => AuthBloc(...)); // wrong


    In this case you need to use UserEntity inside app_user cubit. But app_user
    should not depend on a feature. Therefore we should move the user_entity.dart
    file into "lib/core/entity" folder.




    ---------- (Use state value of one cubit inside another cubit) ----------

        * Never get one cubit to another as dependency in this case.


        * use context.read<Cubit>() in UI layer to read the value and pass them
          to the another cubit as arguments via a function.


    Use BlocListener if you want to call a Cubit method when state of another
    Cubit is changed. Never get one cubit to another as a dependency for this case.

* Dependency injection:

    We use the package: get_it


    Create 2 files called,
        init_dependencies.dart

        final getIt = GetIt.instance;

        void _todos() { // seperate function for each feature
          getIt
            ..registerLazySingleton<TodosDataSource>(
              () => const TodosRemoteDataSource(),

---------- (Some rules to follow) ----------

* Never use HydratedBloc to store secure informations in local storage. Use it
  just to store state related infomations.

      To store HydratedBloc state related data, use "getApplicationSupportDirectory()".
      Use "getApplicationDocumentsDirectory()" only if you want the state to sync
      with cloud backups.


      To store sensitive informations, use "flutter_secure_storage" package.

          Create the singleton class called SecureStorageService and store it in
          "lib/core/service/" directory. Define all read and write logics in this
          class.


      To store non-sensitive data in local storage use suitable storage package.

          Create the singleton class called StorageService and store it in
          "lib/core/service/" directory. Define all read and write logics in this
          class.


* When you need to call a specific methods of a Cubit when a page loads, keep
  them in main page initState method.

      _authCubit.verifyToken();

      _postsCubit.loadPosts();

      _selectedValuesCubit.clear();


      Note:
          If you want to call a method as above, then extract them to seperate
          functions for clean code.


* Always use Container for network images. Because we can provide place holder,
  border, border radius and shape using one widget.


* If any widget is reusable, the extract it to feature/presentation/widget folder.
  If it can be used in multiple features, then extract it to lib/core/widget folder.
  All of other page specific widgets must be defined inside the page.


* Always use "settingsOf(context)" to get arguments to the page. If you use
  ModalRoute.of(context), page will be performed unnecessary rebuilds.

    @override

      super.didChangeDependencies();

---------- Select multiple choices using a cubit -------------------------------

* Cubit example:

    class SelectedServiceOptionsCubit extends Cubit<SelectedServiceOptionsState> {
      SelectedServiceOptionsCubit()
        : super(const SelectedServiceOptionsState.initial());

      void update({
        required int id,
        required int price,
        required int duration,
        bool remove = false,
      }) {
        List<int> optionIds;
        int totalAmount;
        int totalDuration;

        if (remove) {
          optionIds = [...state.optionIds]..remove(id);
          totalAmount = state.totalAmount - price;
          totalDuration = state.totalDuration - duration;
        } else {
          optionIds = [...state.optionIds, id];
          totalAmount = state.totalAmount + price;
          totalDuration = state.totalDuration + duration;
        }

        emit(
          SelectedServiceOptionsState(
            optionIds: optionIds,
            totalAmount: totalAmount,
            totalDuration: totalDuration,
          ),
        );
      }

      void clear() {
        emit(const SelectedServiceOptionsState.initial());
      }
    }




* Extract and seperate the widget which you want to rebuild on each click, and
  wrap it with BlocBuilder.

      state.optionIds.contains(option.id)

      buildWhen: (previous, current) {
        final idInPrevious = previous.optionIds.contains(option.id);
        final idInCurrent = current.optionIds.contains(option.id);

        if (idInPrevious && !idInCurrent) return true;

        if (!idInPrevious && idInCurrent) return true;

        return false;
      }

---------- Create a singleton class --------------------------------------------

* We need a singleton class to make sure that we create and share only one
  client instance inside the application.




---------- (With Eager Initialization) ----------

* Access type:

  1. final socketService = SocketService.instance;

      class SocketService {
        static const instance = SocketService._internal();

        const SocketService._internal();
      }


  2. final socketService = SocketService();

      class SocketService {
        static const _instance = SocketService._internal();

        const SocketService._internal();

        factory SocketService() {
          return _instance;
        }
      }


  Note: By using above techniques we make sure that only one object of
        SocketService class is created.


* Now create the client instance as a private, static and nullable instance variable.

    class SocketService {
      static const instance = SocketService._internal();

      static Socket? _socket;

      const SocketService._internal();
    }


    Note:
      * Why private:
          This variable should not be accessed outside of the class.

      * Why static?
          For memory efficiency.

      * Why nullable?
          If it is not nullable we have to initialize it at the time of
          creation or inside the constructor. Most of the time we have to
          implement some complex logic to initialize the variable. Therefore we
          make it nullable and use a seperate method to initialize it.

      * Why not final?
          We have to re-assign the variable to null at the time of dispose.


* Now create instance methods to initialize, get and dispose the client instance.

  * initialize & get:
      To make sure that the value of the client instance is always not null.


  * dispose:
      To dispose the client instance resources when we don't need it anymore.




    class SocketService {
      static const instance = SocketService._internal();

      static Socket? _socket;

      const SocketService._internal();

      Socket _initializeSocket() {
        final socket = io(
          'http://10.0.2.2:3000',
          OptionBuilder().setTransports(['websocket']).build(),
        );
        return socket.connect();
      }

      Socket get socket {
        if (_socket != null) return _socket!;
        _socket = _initializeSocket();
        return _socket!;
      }

      void dispose() {
        if (_socket == null) return;
        _socket!.dispose();
        _socket = null;
      }
    }




---------- (With Lazy Initialization) ----------

class SocketService {
  static SocketService? _instance;

  const SocketService._internal();

  factory SocketService() {
    if (_instance != null) return _instance!;
    _instance = SocketService._internal();
    return _instance!;
  }
}

* Tip: Create the singleton class object as same as client instance
  (static nullable) and initialize or just return it if not null from factory
  constructor.

---------- Dialogs -------------------------------------------------------------

* Always use custom widgets for dialogs.


Ex:

import 'package:flutter/material.dart';

final class ConfirmationDialog {
  static const instance = ConfirmationDialog._internal();
  static bool _isVisible = false;

  const ConfirmationDialog._internal();

  void close(BuildContext context) {
    if (!_isVisible) return;
    Navigator.pop(context);
    _isVisible = false;
  }

  Future<void> show({
    required BuildContext context,
    required String title,
    required String description,
  }) async {
    if (_isVisible) return;

    await showDialog(
      barrierDismissible: false,
      context: context,
      builder: (context) {
        _isVisible = true;

* We use them to implement a widget like loading dialog.

Ex:

final class _LoadingOverlayController {
  final void Function() hide;
  final void Function(String message) updateMessage;

  const _LoadingOverlayController({
    required this.hide,
    required this.updateMessage,
  });
}

final class LoadingOverlay {
  static const instance = LoadingOverlay._internal();
  static _LoadingOverlayController? _controller;

  const LoadingOverlay._internal();

  _LoadingOverlayController _show({
    required BuildContext context,
    required String message,
  }) {
    final overlay = Overlay.of(context);
    final messageStreamController = StreamController<String>();
    messageStreamController.add(message);
    final width = MediaQuery.of(context).size.width;

    final overlayEntry = OverlayEntry(
      builder: (context) {
        return Material(
          color: Colors.black54,
          child: Center(
            child: Container(
              constraints: BoxConstraints(maxWidth: width - 32),
              padding: const EdgeInsets.symmetric(horizontal: 32, vertical: 16),
              decoration: BoxDecoration(
                color: Colors.teal,
                borderRadius: BorderRadius.circular(8),
              ),
              child: Column(
                mainAxisSize: MainAxisSize.min,
                children: [
                  const SizedBox.square(
                    dimension: 24,
                    child: CircularProgressIndicator(
                      strokeWidth: 3,
                      color: Colors.white,
                    ),
                  ),

                  const SizedBox(height: 24),

                  StreamBuilder(
                    stream: messageStreamController.stream,
                    builder: (context, asyncSnapshot) {
                      return Text(
                        textAlign: TextAlign.center,
                        asyncSnapshot.data ?? '',
                        style: const TextStyle(
                          fontSize: 18,
                          fontWeight: FontWeight.w600,
                          color: Colors.white,
                        ),
                      );
                    },
                  ),

                  const SizedBox(height: 8),

                  const Text(
                    'This may take a few seconds',
                    style: TextStyle(color: Colors.white),
                  ),
                ],
              ),
            ),
          ),
        );
      },
    );

    overlay.insert(overlayEntry);

    return _LoadingOverlayController(
      hide: () {
        overlayEntry.remove();
        overlayEntry.dispose();
        messageStreamController.close();
      },
      updateMessage: (message) {
        messageStreamController.add(message);
      },
    );
  }

  void show({required BuildContext context, required String message}) {
    if (_controller != null) return;
    _controller = _show(context: context, message: message);
  }

  void updateMessage(String message) {
    if (_controller == null) return;
    _controller!.updateMessage(message);
  }

  void hide() {
    if (_controller == null) return;
    _controller!.hide();
    _controller = null;
  }
}

---------- RBAC system (authz.dart) --------------------------------------------

* authz - authorization.


* RBAC: Role Based Access Control.


* This is used to define permissions for each user based on their roles.


* Can use in both frontend and backend.


Ex:

enum Role { admin, user }

abstract interface class Resource {
  // All properties that are related to authz in each resource
  String get ownerID;
}

abstract interface class AccessControl {
  // All properties that are related to authz in the user
  String get id;
  Role get role;

  // All authz action permissions
  bool canDelete(Resource resource);
}


* Implements all resource models like Posts, Products, etc from "Resource"
  interface.

    Ex:

      class Post implements Resource {
        final String ownerID;
        // other properties of the post

        const Post({required this.ownerID});
      }


* Implements User model from "AccessControl" interface.

    Ex:

      class User implements AccessControl {
        final String id;
        final Role role;

        const User({required this.id, required this.role});

        @override
        bool canDelete(Resource resource) {
          switch (role) {
            case Role.admin:
              return true;
            case Role.user:
              return id == resource.ownerID;
          }
        }
      }


* Usage:

    void main() {
      const post = Post(ownerID: '1');
      const user = User(id: '2', role: Role.admin);

      print('Has permission: ${user.canDelete(post)}');
    }

---------- Timer ---------------------------------------------------------------

* Timer is asynchronous.


* Most of the time we create a nullable instance variable to works with a Timer.

    Timer? timer;


* Starts immediately when created and run callback function after the specified
  duration.

    Ex:
      Timer(Duration(seconds: 2), () {
        print('This runs after 2 seconds');
      });


* Starts immediately when created and continuously runs the callback at the
  specified interval.

    Ex:
      Timer.periodic(Duration(seconds: 1), (timer) {
        print('This runs every second');
      });


* timer?.cancel():

    Once you cancel a Timer in Flutter, there's no way to restart or resume it.
    You have to create a new Timer instance.


* timer?.isActive:

    Timer:
        isActive becomes false when,
            * it completes OR

            * when cancelled


    Timer.periodic:
        isActive stays true until you call cancel().


* timer?.tick:

    Number of executions of the Timer.periodic() callback. Not useful for
    Timer().

import 'dart:async';
import 'package:flutter/material.dart';

import 'package:smooth_page_indicator/smooth_page_indicator.dart';

class SlideShow extends StatefulWidget {
  const SlideShow({super.key});

  @override
  State<SlideShow> createState() => _SlideShowState();
}

class _SlideShowState extends State<SlideShow> {
  final _pageController = PageController();
  final _numberOfItems = 5;
  Timer? _timer;
  bool _isReverse = false;
  int _currentIndex = 0;

  @override
  void initState() {
    super.initState();

    _timer = Timer.periodic(const Duration(seconds: 5), (timer) {
      _pageController.animateToPage(
        _currentIndex,
        duration: const Duration(seconds: 1),
        curve: Curves.easeOut,
      );

      if (_currentIndex == _numberOfItems - 1) {
        _isReverse = true;
      } else if (_currentIndex == 0) {
        _isReverse = false;
      }

      if (_isReverse) {
        _currentIndex--;
      } else {
        _currentIndex++;
      }
    });
  }

  @override
  void dispose() {
    _pageController.dispose();
    _timer?.cancel();

    super.dispose();
  }

  @override
  Widget build(BuildContext context) {
    return Column(
      children: [
        SizedBox(
          height: 220,
          child: PageView(
            controller: _pageController,
            children: List.generate(_numberOfItems, (index) {
              return Padding(
                padding: const EdgeInsets.symmetric(horizontal: 16),

---------- Authentication with JWT ------------------------------------------------------

---------- (Online-first application) ----------

* Add following operations in auth feature:
    login
    register
    password reset
    validateToken


* Add following operations to app_user_cubit in
  core/common/cubits/app_user_cubit.dart

    emitAppUserSuccess
    emitAppUserFailure
    signOut




Process:

1. Save jwt in local storage at the initial signup and in every signin.

2. After signin/signup, if the user closes the application and opens it again
   (to auth state persistance),

      * Firstly, check if the jwt is existed in the local storage or not.

      * If not, redirected to LoginPage.

      * If yes, then validate the token from the server.

      * If validation is success, redirected to HomePage.

      * If validation is a failure, redirected to LoginPage.

* (app_router.dart)

    * No need to contain the SplashPage.


    sealed class AppRouter {

      static const loginPage = '/login';
      static const authGate = '/auth-gate';

      static final router = GoRouter(
        initialLocation: authGate,

        routes: [
          GoRoute(path: homePage, builder: (context, state) => const HomePage()),

          GoRoute(path: loginPage, builder: (context, state) => const LoginPage()),

          GoRoute(path: authGate, builder: (context, state) => const AuthGate()),
        ],
      );
    }

    class AuthGate extends StatefulWidget {
      const AuthGate({super.key});

      @override
      State<AuthGate> createState() => _AuthGateState();
    }

    class _AuthGateState extends State<AuthGate> {
      late final AuthCubit _authCubit;

      @override
      void initState() {
        super.initState();

        _authCubit = context.read<AuthCubit>();

        Future.delayed(const Duration(seconds: 2), () {
          _authCubit.validateToken();
        });
      }

      @override
      Widget build(BuildContext context) {
        return BlocListener<AuthCubit, AuthState>(
          listener: (context, state) {
            if (state is AuthSuccess) {
              context.go(AppRouter.homePage);
            } else {
              context.go(AppRouter.loginPage);
            }
          },
          child: const SplashPage(), // or some loading indication
        );
      }
    }

final class SocketClient {
